## Контекст

Существующие данные в interview-to-bpmn низкого качества. Нужно: удалить всё → создать реалистичные входные файлы (текст + аудио) для ПВХ-компании → пропустить через приложение → получить готовые BPMN-диаграммы и Word-документы. Если приложение ломается — чиним код до победного.

## Фаза 2: Создание входных файлов (5 интервью)

### 2.1 Текстовые файлы (.txt) — быстрый путь
Создать 5 файлов в формате диалога (Интервьюер ↔ Респондент), ~3000-5000 слов каждый:

| Файл | Отдел | Респондент | О чём |
|------|-------|-----------|-------|
| `interview_buhgalteria.txt` | Бухгалтерия | Главбух Иванова Е.П. | 1С:Бухгалтерия, учёт сырья, себестоимость, зарплата, двойной ввод |
| `interview_proizvodstvo.txt` | Производство | Нач. цеха Сидоров А.В. | Экструзия ПВХ, план смен, контроль качества, журнал брака |
| `interview_zakupki.txt` | Закупки/Склад | Менеджер Козлова М.С. | ПВХ-смола, стабилизаторы, поставщики, складской учёт Excel |
| `interview_prodazhi.txt` | Продажи | РОП Петров Д.И. | Заказы дилеров, прайс Excel, нет CRM, дебиторка |
| `interview_logistika.txt` | Логистика | Логист Николаев И.К. | Отгрузки, маршруты, ТТН, транспортные компании |

### 2.2 Аудио файлы (.wav) через TTS
Из тех же текстов сгенерировать аудио через edge-tts (Microsoft Azure TTS, бесплатный):
- Голос интервьюера: `ru-RU-DmitryNeural` (мужской)
- Голос респондента: женский/мужской в зависимости от отдела
- Формат: WAV 16kHz mono (то что ожидает faster-whisper)
- Длительность: ~5-10 мин на файл (сокращённая версия для скорости обработки)

## Фаза 3: Обработка текстовых файлов через приложение (быстрый путь)

### 3.1 Импорт .txt → транскрипт JSON
Приложение загружает .txt через `_save_text_as_transcript()` → JSON в `transcripts/`

### 3.2 Извлечение процессов (Ollama)
`extract_processes()` → JSON в `processes/`
- Если Ollama падает/таймаутит → увеличить timeout, добавить retry, исправить парсинг JSON
- Если JSON невалидный → улучшить `_parse_json_response()`, добавить ремонт JSON

### 3.3 Генерация BPMN (Ollama)
`generate_bpmn_json()` → `generate_bpmn_file()` → `render_bpmn()`
- Если BPMN JSON невалидный → улучшить валидацию и retry
- Если SVG некрасивый → исправить layout/renderer

### 3.4 Генерация документов
`generate_documents()` → .docx файлы
- Если ошибка с PNG vs SVG → исправить doc_generator

## Фаза 5: Исправление багов по ходу

Потенциальные проблемы и фиксы:

| Проблема | Файл | Решение |
|----------|------|---------|
| Ollama timeout при длинных текстах | `process_extractor.py` | Увеличить timeout, chunking |
| Невалидный JSON от LLM | `process_extractor.py` | JSON repair (`_parse_json_response`) |
| BPMN JSON не проходит валидацию | `json_to_bpmn.py` | Retry + fallback |
| SVG рендеринг — перекрытие | `layout.py`, `renderer.py` | Улучшить collision resolution |
| Word ищет .png вместо .svg | `doc_generator.py` | Поддержать .svg или сконвертировать |
| Пайплайн виснет на этапе | `pipeline.py` | Timeout + информативные ошибки |
