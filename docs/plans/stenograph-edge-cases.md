# Стенограф — UX Edge Cases и обработка ошибок

> Все крайние случаи, ошибки и нестандартные сценарии.
> Для каждого: что происходит, что видит пользователь, как система реагирует.

---

## 1. Ошибки загрузки и формата

### EC-01: Неподдерживаемый формат файла

| | |
|---|---|
| **Ситуация** | Пользователь перетаскивает `.avi`, `.flv`, `.ogg` или `.txt` |
| **Реакция** | Drop-зона краснеет, сообщение: «Формат не поддерживается. Поддерживаются: MP3, WAV, M4A, MP4, MKV, WEBM» |
| **Действие системы** | Файл не принимается. Без fallback. |

### EC-02: Повреждённый файл

| | |
|---|---|
| **Ситуация** | Файл .mp3 но битый (0 байт, обрезан, неверный заголовок) |
| **Реакция** | Pipeline останавливается на шаге 1. Сообщение: «Не удалось прочитать файл. Файл повреждён или пуст.» |
| **Действие системы** | Статус встречи → `error`. Кнопка «Удалить» / «Загрузить другой файл». |

### EC-03: Очень большой файл (> 2 GB)

| | |
|---|---|
| **Ситуация** | 4-часовое видео в высоком качестве |
| **Реакция** | Предупреждение при загрузке: «Файл {{size}} ГБ. Обработка может занять 15-20 минут. Продолжить?» |
| **Действие системы** | Извлечь аудио (ffmpeg → WAV 16kHz mono), удалить видео-копию после извлечения для экономии места. |

### EC-04: URL недоступен

| | |
|---|---|
| **Ситуация** | Ссылка на YouTube/Google Drive, но видео удалено или приватное |
| **Реакция** | «Не удалось скачать запись. Проверьте ссылку и доступ к файлу.» + показать ошибку от yt-dlp |
| **Действие системы** | Встреча создаётся, но без записи. Можно позже загрузить файл вручную. |

## 2. Ошибки транскрипции

### EC-05: Плохое качество аудио

| | |
|---|---|
| **Ситуация** | Запись с шумом, эхом, далёким микрофоном |
| **Реакция** | Pipeline завершается, но с предупреждением: «Качество транскрипции может быть низким. Обнаружены проблемы: шум, низкая громкость.» |
| **Действие системы** | ffmpeg перед транскрипцией: нормализация громкости + noise reduction. В стенограмме пометки `[неразборчиво]`. |
| **Индикатор** | Иконка «⚠ Низкое качество» на карточке встречи. |

### EC-06: Нет речи в записи

| | |
|---|---|
| **Ситуация** | Файл содержит только музыку/тишину/шум |
| **Реакция** | «В записи не обнаружена речь. Возможно, выбран неверный файл.» |
| **Действие системы** | Статус → `error`. Артефакты не создаются. |

### EC-07: Смешанные языки

| | |
|---|---|
| **Ситуация** | Участники переключаются между русским и английским |
| **Реакция** | Whisper автоматически определяет язык по сегментам. Без специальной обработки. |
| **Действие системы** | В стенограмме сохраняется оригинальный язык каждого сегмента. |

### EC-08: Whisper API timeout / ошибка

| | |
|---|---|
| **Ситуация** | API не отвечает, 429 (rate limit), 500 (server error) |
| **Реакция** | «Ошибка транскрипции. Повторная попытка через 30 секунд...» (автоматический retry) |
| **Действие системы** | Retry 3 раза с exponential backoff (30с → 60с → 120с). После 3 неудач: «Транскрипция не удалась. Проверьте API-ключ и интернет-соединение.» Кнопка «Повторить». |

## 3. Ошибки извлечения артефактов

### EC-09: Claude API ошибка

| | |
|---|---|
| **Ситуация** | API недоступно, невалидный ключ, превышен лимит |
| **Реакция** | Стенограмма уже готова (Whisper отработал). Сообщение: «Транскрипция готова. Извлечение артефактов не удалось: {{error}}. Можно повторить позже.» |
| **Действие системы** | Сохранить стенограмму. Кнопка «Извлечь артефакты» для повторного запуска только этапа извлечения. |

### EC-10: Частичный результат

| | |
|---|---|
| **Ситуация** | 4 из 6 промптов отработали, 2 упали |
| **Реакция** | Показать готовые артефакты. На месте неудавшихся: «Не удалось извлечь. [Повторить]» |
| **Действие системы** | Каждый промпт независим. Успешные сохраняются. Неуспешные можно перезапустить по одному. |

### EC-11: Невалидный JSON от Claude

| | |
|---|---|
| **Ситуация** | LLM вернул текст вместо JSON, или JSON с ошибкой синтаксиса |
| **Реакция** | Тихий retry (до 2 раз). Если 3 раза невалидный → показать сырой текст с пометкой «Не удалось структурировать. Показан сырой результат.» |
| **Действие системы** | Попытка исправить JSON (удалить markdown-обёртки, закрыть скобки). Fallback: показать как plain text. |

### EC-12: Пустой артефакт

| | |
|---|---|
| **Ситуация** | На встрече не обсуждались требования → промпт «Требования» вернул пустой массив |
| **Реакция** | Карточка артефакта серая: «Требований не выявлено в этой встрече» |
| **Действие системы** | Не создавать пустой файл. Карточка неактивна. Нет кнопки экспорта для пустого артефакта. |

## 4. Ошибки проектов и данных

### EC-13: Дублирование записи

| | |
|---|---|
| **Ситуация** | Загрузка того же файла повторно (по имени и размеру) |
| **Реакция** | «Файл с таким именем и размером уже загружен в этот проект (14.03.2026). Загрузить как отдельную встречу?» |
| **Действие системы** | Кнопки: «Заменить существующую» / «Создать новую» / «Отмена». |

### EC-14: Удаление проекта с данными

| | |
|---|---|
| **Ситуация** | Пользователь удаляет проект с 20 встречами |
| **Реакция** | «Удалить проект «Альфа-Логистик»? Это удалит 20 встреч, все артефакты и записи. Введите название проекта для подтверждения:» |
| **Действие системы** | Подтверждение вводом имени (как GitHub). Удаление в корзину (30 дней), не безвозвратное. |

### EC-15: Диск заполнен

| | |
|---|---|
| **Ситуация** | Недостаточно места для сохранения записи или результатов |
| **Реакция** | «Недостаточно места на диске. Требуется: ~{{size}} МБ. Доступно: {{available}} МБ. Освободите место или измените папку хранения в настройках.» |
| **Действие системы** | Проверка свободного места ПЕРЕД загрузкой. Оценка: ~50 МБ на час аудио (сырые + результаты). |

## 5. Ошибки UX и навигации

### EC-16: Обработка уже идёт, пользователь закрывает приложение

| | |
|---|---|
| **Ситуация** | Pipeline на 60%, пользователь закрывает окно |
| **Реакция** | «Обработка записи в процессе (60%). Закрыть приложение? Обработка будет отменена.» |
| **Действие системы** | Кнопки: «Дождаться» / «Закрыть и отменить». При отмене: сохранить частичный результат (стенограмму, если готова). При следующем запуске: «Обработка была прервана. Продолжить с последнего шага?» |

### EC-17: Обработка нескольких файлов одновременно

| | |
|---|---|
| **Ситуация** | Drag & drop 3 файла сразу |
| **Реакция** | Создать 3 встречи, обрабатывать последовательно (не параллельно — API rate limits). Показать очередь. |
| **Действие системы** | Очередь: файл 1 → processing, файл 2 → queued, файл 3 → queued. Прогресс для текущего, счётчик для очереди. |

### EC-18: Очень длинная стенограмма (> 100K токенов)

| | |
|---|---|
| **Ситуация** | 4-часовая встреча, стенограмма > 200K символов |
| **Реакция** | Индикатор: «Длинная запись. Используем разбивку на секции для извлечения.» |
| **Действие системы** | Разбивка стенограммы на чанки по 60K токенов, параллельный запуск промптов для каждого чанка, merge результатов. Может увеличить время и стоимость ×2-3. Предупредить: «Стоимость обработки: ~$2.50» |

### EC-19: Offline (нет интернета)

| | |
|---|---|
| **Ситуация** | Пользователь запускает приложение без интернета |
| **Реакция** | Приложение открывается нормально. Проекты, встречи, артефакты доступны. При попытке загрузить URL или обработать файл: «Нет подключения к интернету. Для транскрипции и извлечения требуется доступ к API.» |
| **Действие системы** | Режим «только просмотр». Редактирование артефактов работает (локально). Экспорт работает. Загрузка файлов работает (сохраняются локально). Обработка — нет. |

## 6. Ошибки API-ключей и настроек

### EC-20: Невалидный API-ключ

| | |
|---|---|
| **Ситуация** | Ключ неверный, истёк, или недостаточно кредитов |
| **Реакция** | При первой попытке обработки: «API-ключ {{provider}} невалиден или недостаточно средств. Проверьте ключ в настройках.» |
| **Действие системы** | Проверка ключей: при вводе в настройках + перед каждой обработкой. Кнопка «Проверить ключ» в настройках (отправляет тестовый запрос). |

### EC-21: Отсутствует API-ключ

| | |
|---|---|
| **Ситуация** | Пользователь пытается обработать запись, но ключи не введены |
| **Реакция** | «Для обработки записей нужны API-ключи. Настроить?» → переход в настройки. |
| **Действие системы** | Блокирует только обработку. Проекты, загрузка файлов, просмотр — работают. |

## 7. Перегенерация и редактирование

### EC-22: Перегенерация при изменённом артефакте

| | |
|---|---|
| **Ситуация** | Пользователь отредактировал протокол, потом нажал «Перегенерировать» |
| **Реакция** | «Вы вносили правки в этот документ. Перегенерация заменит ваши изменения. Создать новую версию или перезаписать?» |
| **Действие системы** | Кнопки: «Новая версия» (сохранить текущую как v2, создать v3) / «Перезаписать» / «Отмена». История версий всегда доступна. |

### EC-23: AI-перегенерация не улучшила

| | |
|---|---|
| **Ситуация** | Пользователь попросил «добавь детали из 23-й минуты», но результат хуже |
| **Реакция** | Показать diff: зелёным новое, красным удалённое. Кнопки: «Принять» / «Отклонить» / «Редактировать вручную» |
| **Действие системы** | При отклонении: откат к предыдущей версии. Ничего не теряется. |

### EC-24: Таймкод ведёт в пустоту

| | |
|---|---|
| **Ситуация** | Клик на таймкод [23:14] в артефакте, но запись удалена или перемещена |
| **Реакция** | «Аудиофайл не найден. Возможно, он был перемещён или удалён.» |
| **Действие системы** | Предложить указать новый путь к файлу. Таймкоды остаются в тексте как информация. |

## 8. Кумулятивная логика

### EC-25: Первая встреча в проекте (нет предыдущих)

| | |
|---|---|
| **Ситуация** | Новый проект, первая запись |
| **Реакция** | Обычная обработка. Все статусы = «new». Кумулятивные документы = артефакты этой встречи. |
| **Действие системы** | Блок «Изменения с прошлых встреч» не показывается. |

### EC-26: Противоречие между встречами

| | |
|---|---|
| **Ситуация** | Встреча 1: «объём 50 документов», Встреча 3: «на самом деле 200» |
| **Реакция** | В кумулятивном ТЗ: желтая метка «⚠ ИЗМЕНЕНО» на требовании. «Было: 50 док/день (10.03). Стало: 200 док/день (14.03).» |
| **Действие системы** | Обе версии сохраняются с историей. Пользователь решает какую оставить. |

### EC-27: Удаление встречи из середины серии

| | |
|---|---|
| **Ситуация** | 5 встреч в проекте, удаляется 3-я |
| **Реакция** | «Удаление встречи повлияет на кумулятивные документы. Данные из этой встречи будут убраны из общего ТЗ, глоссария и карты рисков. Продолжить?» |
| **Действие системы** | Пересчёт кумулятивных артефактов (удалить вклад этой встречи). Или пометить как «данные из удалённой встречи» без удаления из кумулятивных. |

## 9. Стоимость и лимиты

### EC-28: Предупреждение о стоимости

| | |
|---|---|
| **Ситуация** | Перед обработкой длинной записи |
| **Реакция** | Под кнопкой обработки мелким текстом: «~$0.75 (Whisper: $0.36, Claude: $0.39)». Для длинных: «~$2.50 (длинная запись, разбивка на секции)» |
| **Действие системы** | Статистика в настройках: «Потрачено за месяц: $12.40 (16 встреч)» |

### EC-29: Batch-загрузка 20+ файлов

| | |
|---|---|
| **Ситуация** | Пользователь загружает архив встреч за месяц |
| **Реакция** | «Загружено 23 файла. Общая стоимость обработки: ~$17.25. Обработать все?» |
| **Действие системы** | Очередь обработки. Можно выборочно убрать файлы из очереди. Прогресс: «Обработано 5 из 23». Можно остановить очередь в любой момент. |

## 10. Безопасность

### EC-30: API-ключи в хранилище

| | |
|---|---|
| **Ситуация** | Хранение секретов |
| **Реакция** | — |
| **Действие системы** | API-ключи шифруются через Tauri Stronghold (OS-level keychain). Не хранятся в plain text. В UI отображаются как `sk-ant-••••••••`. |

### EC-31: Конфиденциальные данные в стенограммах

| | |
|---|---|
| **Ситуация** | На встрече обсуждаются зарплаты, пароли, коммерческая тайна |
| **Реакция** | Данные отправляются в API (Whisper/Claude). Предупреждение при первом запуске: «Записи отправляются в облачные API для обработки. Убедитесь, что это допустимо для ваших данных.» |
| **Действие системы** | Ничего не кешируется на стороне API (по умолчанию Anthropic не использует данные для обучения). Локальные данные хранятся только на ПК пользователя. |
