## 2. Результаты интервью

### Контекст использования
- **Типы встреч:** все — обследование, рабочие, демонстрации, приёмки
- **Объём:** зависит от фазы (обследование: 20+/мес, поддержка: 3-5/мес)
- **Текущий процесс:** микс — критичные расшифровывает вручную (2-3 ч на встречу), остальные теряются
- **Боль:** потеря ценной информации из «неважных» встреч, рутинная ручная работа

### Входные данные
- **Форматы:** аудио (MP3/WAV), видео (MP4/MKV), онлайн-ссылки (YouTube, Google Drive)
- **Длительность:** от 10 минут до 4+ часов
- **Язык:** русский (с техническими терминами на английском)

### Выходные артефакты (полный пакет)

| # | Артефакт | Описание |
|---|----------|----------|
| 1 | **Протокол встречи** | Участники, повестка, решения, задачи с ответственными и сроками |
| 2 | **ТЗ / Требования** | FR/NFR, описание процессов AS-IS, ограничения |
| 3 | **Стенограмма** | Полный текст с таймкодами и разметкой спикеров |
| 4 | **Карта рисков** | Выявленные риски, неопределённости, противоречия |
| 5 | **Глоссарий терминов** | Словарь предметной области заказчика |
| 6 | **Список открытых вопросов** | Что не договорили, что уточнить на следующей встрече |

### Ключевые требования
- **Проекты:** критична работа с серией встреч (10-30 на проект), накопительная логика
- **Редактирование:** встроенный редактор + перегенерация через AI с уточнениями
- **Экспорт:** DOCX (Word) — основной формат
- **Пользователь:** один (без авторизации, ролей, серверной части)

## 3. Архитектура

### 3.1. Стек технологий

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| **Desktop shell** | Tauri 2.x (Rust) | 5 MB установщик, 30-50 MB RAM, нативная скорость |
| **UI** | React 19 + Tailwind CSS 4 + Framer Motion | Знакомый стек, мощные анимации |
| **Состояние** | Zustand | Лёгкий, без boilerplate |
| **Редактор** | TipTap (ProseMirror) | Rich-text, расширяемый, Markdown-совместимый |
| **БД** | SQLite (через Tauri) | Метаданные, индекс, проекты |
| **Файлы** | Локальная FS | Записи, транскрипты, артефакты |
| **AI: Транскрипция** | OpenAI Whisper API | Лучшее качество для русского языка |
| **AI: Извлечение** | Claude API (Sonnet 4.6) | Лучшее понимание контекста, структурирование |
| **AI: Speaker ID** | OpenAI Audio API | Разделение спикеров |
| **Аудио** | ffmpeg (bundled) | Извлечение аудио из видео, нормализация |
| **Экспорт** | docx.js | Генерация DOCX на клиенте |
| **Скачивание** | yt-dlp (bundled) | Загрузка из YouTube, Google Drive |

### 3.3. AI Pipeline (детально)

```
Вход: файл / URL
    │
    ▼
[1] ЗАГРУЗКА
    ├── Локальный файл → копировать в /projects/{id}/recordings/
    ├── URL → yt-dlp → скачать → сохранить
    └── Видео → ffmpeg → извлечь аудиодорожку (.wav 16kHz mono)
    │
    ▼
[2] РАЗБИВКА (для записей > 30 мин)
    └── ffmpeg → чанки по 25 мин с перекрытием 30 сек
    │
    ▼
[3] ТРАНСКРИПЦИЯ
    └── OpenAI Whisper API (model: whisper-1)
        ├── Язык: ru
        ├── Формат: verbose_json (с таймкодами)
        └── Результат: segments[] с start, end, text
    │
    ▼
[4] СБОРКА СТЕНОГРАММЫ
    ├── Объединение чанков (с дедупликацией перекрытий)
    ├── Speaker diarization (если доступно)
    └── Результат: transcript.json с полной стенограммой
    │
    ▼
[5] ИЗВЛЕЧЕНИЕ АРТЕФАКТОВ (параллельно, Claude API)
    │
    ├── Промпт "Протокол встречи"
    │   → participants, agenda, decisions[], tasks[], deadlines
    │
    ├── Промпт "Требования"
    │   → functional_requirements[], non_functional[], constraints[]
    │
    ├── Промпт "Карта рисков"
    │   → risks[], uncertainties[], contradictions[]
    │
    ├── Промпт "Глоссарий"
    │   → terms[{term, definition, context, timestamp}]
    │
    └── Промпт "Открытые вопросы"
        → questions[], topics_to_clarify[], missing_info[]
    │
    ▼
[6] КОНТЕКСТ ПРОЕКТА (для серии встреч)
    ├── Загрузить предыдущие артефакты проекта
    ├── Сравнить с новыми: что изменилось, что противоречит
    ├── Обновить кумулятивные документы (ТЗ, глоссарий)
    └── Отметить эволюцию решений
    │
    ▼
[7] СОХРАНЕНИЕ
    ├── artifacts/{meeting_id}/*.json — структурированные данные
    ├── Индексация в SQLite (полнотекстовый поиск)
    └── UI-уведомление: "Обработка завершена"
```

### 3.4. Модель данных

```
Project
├── id: uuid
├── name: string
├── description: string
├── created_at: datetime
├── updated_at: datetime
├── settings: ProjectSettings
│   ├── meeting_type_default: enum
│   ├── artifact_templates: string[]
│   └── export_format: "docx"
│
├── meetings: Meeting[]
│   ├── id: uuid
│   ├── title: string (авто или ручное)
│   ├── date: datetime
│   ├── duration_seconds: int
│   ├── source_type: "file" | "url"
│   ├── source_path: string
│   ├── meeting_type: "обследование" | "рабочая" | "демонстрация" | "приёмка" | "другое"
│   ├── status: "uploaded" | "processing" | "transcribing" | "extracting" | "done" | "error"
│   ├── progress: 0-100
│   │
│   ├── transcript: Transcript
│   │   ├── segments: Segment[]
│   │   │   ├── start: float (секунды)
│   │   │   ├── end: float
│   │   │   ├── text: string
│   │   │   └── speaker: string | null
│   │   └── full_text: string
│   │
│   └── artifacts: Artifact[]
│       ├── id: uuid
│       ├── type: "protocol" | "requirements" | "risks" | "glossary" | "questions"
│       ├── title: string
│       ├── content: JSON (структура зависит от типа)
│       ├── content_markdown: string (для редактора)
│       ├── version: int
│       ├── created_at: datetime
│       └── edited_at: datetime
│
└── cumulative_artifacts: CumulativeArtifact[]
    ├── type: "requirements" | "glossary" | "risks"
    ├── content: JSON (объединённые данные всех встреч)
    ├── source_meetings: uuid[]
    └── last_updated: datetime
```

## 5. Пользовательские сценарии

### 5.1. Первый запуск

```
1. Установка: скачал .msi, установил (5 MB)
2. Первый запуск → Экран приветствия с анимацией
3. Ввод API-ключей (Claude + OpenAI)
4. Выбор папки для хранения данных
5. Готово → Dashboard (пустой, с подсказкой "Создай первый проект")
```

### 5.2. Быстрая обработка записи

```
1. Dashboard → [+ Новый проект] → "Альфа-Логистик"
2. Проект → Drag & Drop аудиофайл (или кнопка + вставка URL)
3. Выбор типа встречи: "Предпроектное обследование"
4. Pipeline запускается автоматически
5. Наблюдаю прогресс в реальном времени (предварительный текст)
6. ~5 мин → "Обработка завершена" → анимация появления карточек
7. Клик на "Протокол" → просмотр и редактирование
8. Клик на таймкод → прослушивание фрагмента
9. [⬇ DOCX] → скачал протокол в Word
```

### 5.3. Серия встреч

```
1. Проект "Альфа-Логистик" уже содержит 5 встреч
2. Загружаю 6-ю запись → Pipeline
3. На этапе "Контекст проекта" система:
   - Сравнивает новые требования с предыдущими
   - Находит: "Требование FR-03 изменено (было X, стало Y)"
   - Обновляет кумулятивный глоссарий (+4 новых термина)
   - Добавляет 2 новых риска в общую карту
4. На странице встречи: блок "Изменения с прошлых встреч"
5. В сайдбаре: кумулятивные документы обновились
```

### 5.4. Перегенерация с уточнением

```
1. Открыл артефакт "Требования"
2. Раздел "NFR" слишком общий
3. Клик [🔄 AI] → поле ввода
4. Пишу: "Детализируй NFR по производительности,
   учти обсуждение на 34-й минуте про 1000 документов в день"
5. AI перегенерирует раздел → diff-подсветка изменений
6. Принимаю / отклоняю / редактирую вручную
7. Version +1 (история версий сохраняется)
```

### 5.5. Глобальный поиск

```
1. Dashboard → строка поиска → "маршрутизация"
2. Результаты из всех проектов с таймкодами
3. Клик на результат → переход в артефакт на нужный фрагмент
4. Клик на таймкод → прослушивание с 30 сек до и после
```
