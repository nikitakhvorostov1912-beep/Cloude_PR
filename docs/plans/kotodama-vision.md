# Kotodama (言霊) — Vision Document

> «Дух слов» — AI-приложение для трансформации аудио/видеозаписей встреч в структурированные артефакты.

## 2. Дизайн-система

### 2.1 Палитра «Midnight Amber»

```
Тёмная тема (основная):
┌─────────────────────────────────────────────┐
│ Background:  #0B0E1A  (midnight blue)       │
│ Surface:     #131726  (dark surface)        │
│ Accent1:     #E5A336  (amber — основной)    │
│ Accent2:     #CC8832  (dark amber)          │
│ Accent3:     #8B7355  (matte bronze)        │
│ Highlight:   #F5D280  (light amber)         │
│ Subtle:      #2A3050  (muted blue)          │
│ Text:        #E8E0D4  (warm white)          │
│ TextSecond:  #8890A8  (muted text)          │
└─────────────────────────────────────────────┘

Светлая тема: инверсия тёмной
Фон: градиентные blob-ы (парящие цветные пятна под стеклом)
```

### 2.2 Стиль

| Элемент | Решение |
|---------|---------|
| Парадигма | Premium Glassmorphism + 3D |
| Стекло | Средний уровень: карточки, сайдбар, панели, кнопки |
| Кнопки | Glass-кнопки (полупрозрачные blur + градиентные бордеры) |
| Иконки | Кастомные 3D-иконки |
| Типографика | Микс: элегантные заголовки + чистый sans body + mono code |
| 3D-эффекты | Tilt/parallax + Three.js сцены + 3D-переходы |
| Звуки | Опциональные (отключаемые в настройках) |
| Анимации | WOW-уровень: максимум |
| Тема | Обе (переключаемые dark/light) |

### 2.3 Микро-взаимодействия

| Элемент | Анимация |
|---------|----------|
| DnD загрузка | Портал/воронка (файл засасывается в 3D-портал) |
| Уведомления | Glass-панель сверху (macOS-стиль) |
| Пустые состояния | 3D-иллюстрация + CTA-кнопка |
| Переходы страниц | Shared element transitions |

## 3. Навигация и экраны

### 3.1 Навигация

**Bottom Dock** в стиле macOS:
- Эффект увеличения при наведении
- Bounce-анимация при клике
- Индикаторы активного экрана
- Индикаторы уведомлений

### 3.2 Экраны (8 штук)

| # | Экран | Описание | MVP |
|---|-------|----------|-----|
| 1 | **Onboarding** | 3D-интро → возможности → API → тема → первая запись (5 шагов) | Да |
| 2 | **Dashboard** | DnD-зона + список проектов по папкам/категориям | Да |
| 3 | **Project** | Детали проекта + файлы + результаты | Да |
| 4 | **Pipeline** | 3D-визуализация обработки (Three.js) | Да |
| 5 | **Viewer** | Единая страница с якорной навигацией (все артефакты со скроллом) | Да |
| 6 | **Templates** | Предустановленные + кастомные профили артефактов | Да |
| 7 | **Settings** | API-ключи, тема, язык, лимит файлов, папка хранения | Да |
| 8 | **Analytics** | Метрики (определить позже) | Нет |

### 3.3 Окно

- Тип: адаптивное
- Минимальное разрешение: 1024x768

## 4. Функциональность

### 4.1 Загрузка файлов

| Параметр | Значение |
|----------|---------|
| Способ | DnD + файловый диалог (с анимацией портала) |
| Форматы | Всё что поддерживает ffmpeg (аудио + видео) |
| Лимит размера | Настраиваемый пользователем |
| Очередь | Последовательная обработка, отображение в карусели карточек |
| Live-запись | Нет (только файлы) |
| Waveform | Да, интерактивная (клик для перехода к моменту) |
| Качество аудио | Авто-улучшение (нормализация, шумоподавление через ffmpeg) |
| Стоимость API | Показывать оценку перед запуском обработки |

### 4.2 AI-пайплайн

```
Upload → Extract Audio (ffmpeg) → Normalize → Chunk →
→ Transcribe (Whisper API) → Diarize (спикеры) →
→ Generate Artifacts (Claude) → Export
```

| Компонент | Технология |
|-----------|-----------|
| STT | OpenAI Whisper API |
| LLM | Только Claude (Anthropic) |
| Диаризация | Да, с автоопределением имён + ручное редактирование |
| Отображение спикеров | Цветовые блоки (каждый спикер = свой цвет) |
| Аудио-извлечение | ffmpeg |

### 4.3 Артефакты

| Параметр | Значение |
|----------|---------|
| Предустановленные | 3 базовых: Протокол, Стенограмма, Action Items |
| Кастомные | Пользователь создаёт свои профили |
| Выбор | Профили + кастом (перед запуском) |
| Просмотр | Только чтение (без редактирования) |
| Экспорт | DOCX + PDF |
| Перезапуск | Полный (с другим профилем/настройками) |
| Стриминг | Да, промежуточные результаты в реальном времени |

### 4.4 Организация проектов

| Параметр | Значение |
|----------|---------|
| Структура | Папки/Категории |
| Статусы | Нет |
| Корзина | Нет (удаление с подтверждением, удаляет всё) |
| Фильтры | Расширенные: дата, теги, длительность, участники |
| Удаление | Полное (метаданные + исходные файлы + результаты) |

### 4.5 Pipeline-экран

| Параметр | Значение |
|----------|---------|
| Визуализация | 3D-сцена (Three.js) с анимацией обработки |
| Фоновая обработка | Да, с уведомлением |
| Ошибки | Подробная диагностика + предложения по исправлению |
| Промежуточные результаты | Стриминг текста |

## 7. Roadmap

### Подход
- **MVP**: Полный (всё кроме аналитики)
- **Приоритет**: Дизайн + функционал параллельно
- **Этапы**: 5+ итераций (маленькие релизы)
- **Backend**: Полный рерайт с нуля
- **Первый экран**: Onboarding

### Предложенные итерации

```
Итерация 1: Фундамент
  ├─ Дизайн-система (палитра, компоненты, glass-эффекты)
  ├─ Tauri scaffold + SQLite + Stronghold
  ├─ Роутинг + Bottom Dock навигация
  └─ Onboarding (5 шагов, 3D-интро, маскот)

Итерация 2: Ядро загрузки
  ├─ Dashboard (DnD-зона с порталом, список проектов)
  ├─ Проектная система (папки, CRUD, фильтры)
  ├─ Загрузка файлов (ffmpeg извлечение, нормализация)
  └─ Waveform-визуализация

Итерация 3: AI-пайплайн
  ├─ Pipeline-экран (3D-визуализация обработки)
  ├─ Whisper API интеграция (транскрипция + диаризация)
  ├─ Claude интеграция (генерация артефактов)
  ├─ Стриминг промежуточных результатов
  └─ Оценка стоимости API

Итерация 4: Результаты + Экспорт
  ├─ Viewer (единая страница, цветовые блоки спикеров)
  ├─ Экспорт DOCX + PDF
  ├─ Перезапуск обработки
  ├─ Очередь файлов (карусель)
  └─ Фоновая обработка + уведомления

Итерация 5: Шаблоны + Полиш
  ├─ Templates (3 предустановленных + кастомные)
  ├─ Settings (полный функционал)
  ├─ Backup (Export/Import)
  ├─ Звуковые эффекты
  └─ Финальная полировка анимаций

Позже:
  ├─ Analytics экран
  ├─ History/Search
  ├─ Авто-обновление
  └─ Монетизация
```
