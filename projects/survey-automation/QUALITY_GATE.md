# Quality Gate - Чек-лист приёмки

Проект считается готовым только при выполнении ВСЕХ пунктов.

## 1. Запуск
- [ ] `start.bat` запускает backend + frontend без ошибок
- [ ] http://localhost:3000 открывается в браузере
- [ ] http://localhost:8000/docs доступна (Swagger UI)

## 2. Backend API
- [ ] Все endpoints отвечают корректно
- [ ] `pytest tests/ -v` — все тесты зелёные
- [ ] Обработка ошибок на каждом endpoint (4xx, 5xx)
- [ ] Pydantic модели для всех запросов/ответов

## 3. Frontend Build
- [ ] `npm run build` без ошибок и предупреждений
- [ ] Нет TypeScript ошибок
- [ ] Нет неиспользуемых импортов

## 4. UI Функциональность
- [ ] Все кнопки кликабельны и выполняют действие
- [ ] Все формы валидируются и отправляются
- [ ] Навигация работает (сайдбар, ссылки, кнопки назад)
- [ ] Нет заглушек, TODO, placeholder-контента
- [ ] Все loading-состояния отображаются
- [ ] Все error-состояния отображаются

## 5. Пайплайн
- [ ] Создание проекта работает
- [ ] Загрузка файлов работает (drag-drop + выбор)
- [ ] Стадия 1: транскрипция запускается
- [ ] Стадия 2: извлечение процессов работает
- [ ] Стадия 3: BPMN генерация работает
- [ ] Стадия 4: GAP-анализ работает
- [ ] Стадия 5: TO-BE генерация работает
- [ ] Стадия 6: генерация документов работает
- [ ] Прогресс-бар обновляется

## 6. Экспорт
- [ ] Visio (.vsdx) генерируется и скачивается
- [ ] Word описания процессов генерируется
- [ ] Excel требования генерируется
- [ ] Word требования генерируется
- [ ] GAP-анализ Excel генерируется
- [ ] ZIP со всеми файлами генерируется

## 7. UX/Дизайн
- [ ] Тёмная тема по умолчанию
- [ ] Все тексты на русском языке
- [ ] Стиль как у Claude app (минималистичный, чистый)
- [ ] Адаптивность (не ломается при изменении размера)
- [ ] Нет визуальных багов (перекрытие, обрезание)

## 8. Тесты
- [ ] Backend unit тесты проходят
- [ ] Backend integration тесты проходят
- [ ] Frontend E2E тесты проходят (Playwright)

## 9. Консоль
- [ ] Нет `console.error` в браузере
- [ ] Нет необработанных exception в backend логах
- [ ] Нет предупреждений React (StrictMode)

## 10. Тестовые данные
- [ ] Тестовый проект создан и проходит весь пайплайн
- [ ] Тестовые транскрипты реалистичны
- [ ] Все экспортные файлы корректны

## Вердикт

**PASS** — все пункты выполнены
**FAIL** — есть невыполненные пункты (перечислить)
