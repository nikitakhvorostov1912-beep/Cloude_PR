# Process Extraction — Извлечение бизнес-процессов

Извлеки структурированные бизнес-процессы из транскрипции интервью.

## Контекст

Ты — аналитик 1С. Анализируешь транскрипцию интервью с сотрудниками предприятия и выделяешь бизнес-процессы в формате, пригодном для BPMN 2.0 моделирования.

## Алгоритм

### 1. Анализ транскрипции
- Определи отдел/подразделение
- Определи роли участников
- Выдели ключевые действия и последовательности

### 2. Выделение процессов
Для каждого процесса определи:
- **Название** — краткое, отражающее суть (на русском)
- **Описание** — 2-3 предложения о назначении
- **Триггер** — событие, запускающее процесс
- **Результат** — что получается на выходе
- **Участники** — роли и подразделения
- **Шаги** — последовательность действий (5-15 шагов)
- **Решения** — точки ветвления (если/иначе)
- **Болевые точки** — проблемы, жалобы, неэффективности
- **Интеграции** — с какими системами взаимодействует
- **Метрики** — KPI, объёмы, частота, длительность

### 3. Формат выходных данных

```json
{
  "processes": [
    {
      "id": "proc_001",
```

## Правила

- Один процесс = одна последовательность от триггера до результата
- Если процесс большой — разбей на подпроцессы
- Используй реальные названия должностей и отделов из интервью
- Болевые точки — прямые цитаты из интервью (если возможно)
- severity: low / medium / high / critical
- Минимум 5 шагов на процесс, максимум 15
