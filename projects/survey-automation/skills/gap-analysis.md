# GAP Analysis — GAP-анализ под конфигурацию 1С

Проведи GAP-анализ бизнес-процессов в сравнении с типовым функционалом выбранной конфигурации 1С.

## Методика

### Входные данные
- AS-IS процессы (JSON из /extract-processes)
- Выбранная конфигурация 1С (ERP / КА / УТ / ЗУП / БП)
- Справочник модулей конфигурации (app/analysis/erp_capabilities.py)

### Алгоритм оценки

Для каждого процесса / шага:
1. Определи какой модуль 1С покрывает этот функционал
2. Оцени степень покрытия (0-100%)
3. Опиши разрыв (что не покрывается)
4. Оцени трудозатраты на закрытие разрыва
5. Дай рекомендацию

### Шкала покрытия
| % | Статус | Описание |
|---|--------|----------|
| 90-100 | Полное | Типовой функционал закрывает потребность |
| 70-89 | Частичное+ | Незначительные доработки / настройка |
| 50-69 | Частичное | Существенные доработки |
| 30-49 | Минимальное | Значительная разработка |
| 0-29 | Отсутствует | Требуется разработка или интеграция |

### Оценка трудозатрат
- **Настройка** (1-8ч) — типовые настройки, справочники, права
- **Доработка** (8-40ч) — изменение форм, отчётов, печатных форм
- **Разработка** (40-160ч) — новые объекты, подсистемы, обработки
- **Интеграция** (40-200ч) — связь с внешними системами

  "config_name": "1С:ERP",
  "gaps": [
    {
      "process_id": "proc_001",

      "gap_description": "Типовой функционал покрывает основной цикл. Не хватает автоматической проверки кредитного лимита клиента.",

      "effort_hours": 16,
      "recommendation": "Настроить типовые документы Заказ клиента → Реализация. Доработать проверку кредитного лимита через дополнительную обработку.",

## Правила
- Используй реальные названия документов и отчётов 1С
- Трудозатраты в человеко-часах
- Рекомендации конкретные (какой документ настроить, что доработать)
- Сводка с общей статистикой обязательна
