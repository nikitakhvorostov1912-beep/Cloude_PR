# Visio Generation — Генерация Visio диаграмм

Генерируй Visio (.vsdx) файлы из структурированных BPMN JSON данных.

## Библиотека: vsdx

### Подход
Библиотека `vsdx` работает с .vsdx файлами (ZIP с XML внутри). Основной подход:
1. Принять BPMN JSON с элементами и потоками
2. Рассчитать layout (позиции элементов)
3. Создать Visio документ с фигурами и коннекторами
4. Организовать swim lanes для участников

### BPMN → Visio маппинг

| BPMN элемент | Visio фигура | Визуализация |
|-------------|-------------|--------------|
| StartEvent | Круг (тонкий) | Зелёный круг |
| EndEvent | Круг (толстый) | Красный круг |
| UserTask | Прямоугольник | С иконкой пользователя |
| ServiceTask | Прямоугольник | С иконкой шестерёнки |
| ManualTask | Прямоугольник | С иконкой руки |
| ExclusiveGateway | Ромб | С "X" внутри |
| ParallelGateway | Ромб | С "+" внутри |
| InclusiveGateway | Ромб | С "O" внутри |
| SubProcess | Прямоугольник | С "+" внизу |
| IntermediateTimer | Двойной круг | С иконкой часов |
| IntermediateMessage | Двойной круг | С иконкой конверта |
| SequenceFlow | Стрелка | Сплошная линия |
| Lane | Горизонтальная полоса | С названием слева |
| Pool | Контейнер | Объединяет lanes |

### Размеры (в мм)
- Событие: 12x12 мм
- Задача: 40x25 мм
- Шлюз: 15x15 мм
- Lane высота: 60 мм
- Горизонтальный отступ: 25 мм
- Вертикальный отступ: 20 мм

### Layout алгоритм
1. Определи количество lanes (по участникам)
2. Расположи lanes вертикально
3. Элементы в каждой lane слева направо по порядку выполнения
4. Шлюзы создают вертикальные ветвления
5. Коннекторы маршрутизируются автоматически

## Правила
- Русский текст во всех элементах
- Читаемый шрифт (не менее 8pt)
- Страница A3 альбомная для больших процессов
- Цветовая схема: нейтральные тона, акценты для событий
- Каждый процесс — отдельная страница в Visio
