# UX/UI аудит: Aether
Дата: 2026-03-20

## Общее впечатление

Aether имеет хорошо спроектированный главный сценарий (Upload → Pipeline → Artifacts) и приятную glassmorphism-эстетику. Главная проблема — навигация избыточна: 7 пунктов в сайдбаре при реально нужных 4–5. Страниц слишком много для текущего объёма функциональности, что создаёт ощущение пустоты в ряде разделов. Тема — только светлая, что противоречит позиционированию продукта как премиального (dark glassmorphism по всему рынку).

## Навигация

### Sidebar (src/components/layout/Sidebar.tsx)

7 пунктов навигации: Главная, Проекты, Обработка, Артефакты, Шаблоны, Справка, Настройки.

**Проблема: слишком много пунктов для v0.1.**
- "Главная" (/) дублирует функцию DragDropZone — ту же зону загрузки есть и на DashboardPage и на PipelinePage
- "Обработка" и "Артефакты" — это два шага одного сценария. Отдельный пункт "Артефакты" в сайдбаре дублирует переход
- "Справка" занимает постоянное место в навигации — слишком высокий приоритет для вспомогательного контента

**Предлагаемая структура сайдбара (5 пунктов):**
```
Главная    → DashboardPage (quick actions + recent)
Проекты   → ProjectsListPage + ProjectPage
Обработка → PipelinePage (Upload → Pipeline → Results встроены)
Настройки → SettingsPage
?         → Справка — иконка "?" внизу сайдбара (не как основной пункт)
```
Шаблоны — убрать из сайдбара, встроить в PipelinePage как inline-выбор.

### Роуты (src/App.tsx)

9 маршрутов при 7 пунктах сайдбара. Нет lazy loading — все 9 страниц импортируются синхронно.

### OnboardingPage (src/pages/OnboardingPage.tsx)

3 шага: Welcome → API Keys → Ready.

**Сильные стороны:**
- Правильная логика "free-first": первым показывается Groq (бесплатный)
- Прогресс-индикатор с анимацией
- Шаг 2 объясняет что даёт каждый ключ (бейдж "Бесплатно", лимиты)
- Валидация формата ключа в реальном времени (`gsk_` паттерн)

**Слабые стороны:**
- Шаг 1 (Welcome) слишком минималистичен, нет упоминания что обработка локальная/приватная
- Шаг 2: нет ссылки на console.groq.com для получения ключа
- Шаг 3 пустой — только галочка. Можно использовать для быстрого ввода контекста
- Нет возможности skip для продвинутых пользователей

## Pipeline UX

### PipelinePage (src/pages/PipelinePage.tsx, 699 строк)

**До запуска:**
- DragDropZone + кнопка "Обработать" — чисто и понятно
- Три информационных карточки (Проект, Шаблон, Провайдер) — полезно
- **UX-проблема:** кнопка "Обработать" появляется только после добавления файлов — при первом взгляде непонятно что делать

**Во время обработки:**
- Прогресс-бар + процент в заголовке — дублирование
- Три компонента прогресса одновременно: PipelineStages + ArtifactProgress + StreamingText — **перегрузка визуальных сигналов**

**После завершения:**
- Карточка успеха с двумя кнопками — правильно
- Навигация к артефактам корректная

### Компоненты прогресса — дублирование

**PipelineStages:** прогресс-бар + 5 карточек этапов (Upload, Extract, Transcribe, Generate, Complete)

**ArtifactProgress:** сетка карточек артефактов. Визуально схожа со списком этапов — путаница

**Вердикт:** PipelineStages и ArtifactProgress не дублируют смысл, но визуально перегружают экран. Рекомендуется: верхняя часть — горизонтальный stepper, нижняя — сетка артефактов.

**StreamingText:** накапливается без лимита (P1-5). Длинный текст выталкивает кнопки вниз.

### DragDropZone (src/components/upload/DragDropZone.tsx, 337 строк)

**Не перегружена.** Хорошо разделяет Tauri и web-режим, поддерживает multi-file, drag-reorder.

**Замечания:**
- Кнопки ▲/▼ (`w-5 h-5` = 20px) — ниже минимального touch target 44px
- play('upload') и play('success') вызываются подряд при drop — два звука за один жест
- Emoji иконки файлов (🎥/🎵) — непоследовательность на разных ОС

## Артефакты

### Типы (src/types/artifact.types.ts)

8 типов: `protocol`, `requirements`, `risks`, `glossary`, `questions`, `transcript`, `development`, `custom`.

**Все 8 типов нужны и логично именованы.** Единственный спорный: `custom` — почти заглушка (нет UI для создания кастомного промпта при запуске пайплайна).

### ArtifactViewer (src/components/artifacts/ArtifactViewer.tsx)

Чистый switch-компонент. 110 строк — компактный. Цепочка ViewerPage → ArtifactViewer → *View — глубина 3 уровня, **нормальная архитектура**.

**Мёртвый компонент:** ArtifactTabs.tsx существует, нигде не импортируется (подтверждено).

### ViewerPage

Два состояния: список встреч (без meetingId) и просмотр артефактов (с meetingId). Правильный паттерн URL-driven. Но нет breadcrumb "← Все встречи" при просмотре.

## Настройки

### SettingsPage (src/pages/SettingsPage.tsx)

**8 провайдеров LLM+STT на одной странице** — перегруженно.

**UX-проблемы:**

1. **Пользователь не понимает какой провайдер выбрать.** Нет сравнительного guide. Для непрофессионала "Cerebras", "Mistral", "OpenRouter" ничего не говорят.

2. **Связь "провайдер → ключ" неявная.** При выборе провайдера нет автоскролла к полю ключа.

3. **LLM и STT разделены.** Groq присутствует в обоих — путаница.

4. **Порядок секций.** "Режим маршрутизации" — первый элемент, хотя это продвинутая опция.

**Рекомендуемый порядок:**
```
1. API-ключи (главное)
2. Провайдеры (выбор после ввода ключей)
3. Режим маршрутизации (продвинутые настройки)
4. Звук / Другое
```

5. **8 полей ввода паролей** — визуально тяжело. Показывать только настроенные + кнопку "Добавить ключ".

## Что убрать

### GuidePage — ТРАНСФОРМИРОВАТЬ
Содержит качественный контент (описания артефактов, советы, рабочий процесс). Но отдельная страница "Справка" в основной навигации — неправильное место.

**Рекомендация:**
- Описания артефактов → tooltip/popover в PipelinePage
- Советы по качеству аудио → collapsed hint в DragDropZone
- Рабочий процесс → в онбординг
- Кумулятивный контекст → tooltip в ProjectPage
- Саму страницу оставить как `/guide`, убрать из сайдбара

### TemplatesPage — УПРОСТИТЬ
Нужна для CRUD шаблонов, но не должна быть в основной навигации.

**Рекомендация:**
- Inline-селектор шаблона встроить в PipelinePage (модалка/drawer вместо навигации)
- CRUD шаблонов доступен через PipelinePage → "Изменить шаблон"

### ArtifactTabs.tsx — УДАЛИТЬ
Мёртвый компонент.

## Сравнение с конкурентами

### Что у конкурентов лучше
1. **Waveform + текст синхронизация** (Buzz, HumanSignal) — клик по timestamp → переход в аудио
2. **Горизонтальный stepper** (MacWhisper, Vibe) — компактнее вертикального списка
3. **Контекстные tooltips вместо справки** (Fathom) — удобнее отдельной страницы
4. **Быстрый старт без API-ключа** (MacWhisper) — локальные модели
5. **Тёмная тема как стандарт** (Vibe, MacWhisper, Voibe) — Aether единственный со светлой

### Что перенять

| Паттерн | От кого | Приоритет |
|---------|---------|-----------|
| Тёмная тема | Vibe, MacWhisper | КРИТИЧНО |
| Горизонтальный stepper прогресса | MacWhisper, Vibe | ВЫСОКИЙ |
| Контекстные tooltips | Fathom | ВЫСОКИЙ |
| Waveform + timestamped transcript | Buzz, HumanSignal | СРЕДНИЙ |
| Keyboard shortcuts | Buzz, все pro-продукты | СРЕДНИЙ |
| Batch processing queue | Vibe, MacWhisper | НИЗКИЙ |
