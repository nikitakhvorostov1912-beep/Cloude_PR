# Aether — Product Requirements Document

> Версия 0.1.0 | Март 2026

## 1. Executive Summary

### Проблема

IT-консультанты, аналитики и руководители проектов тратят 2-4 часа на ручную обработку каждой записи встречи: расшифровка, выделение решений, формулирование требований, выявление рисков. При 5-10 встречах в неделю это 10-40 часов потерянного времени. Существующие инструменты транскрипции дают «сырой» текст, который всё равно требует ручной структуризации.

### Решение

Aether — десктопное приложение для Windows, которое автоматически превращает аудио/видеозаписи встреч в 6 типов структурированных проектных артефактов с помощью AI (OpenAI Whisper + Claude/GPT-4). Устанавливается инсталлятором (~2 MB), работает локально, API-ключи хранятся в зашифрованном хранилище.

### Ключевые метрики (KPI)

| KPI | Цель | Метод измерения |
|-----|------|-----------------|
| Время обработки | < 5 мин для 1 часа записи | Таймер в pipeline |
| Точность артефактов | > 85% (валидация пользователем) | Обратная связь |
| Стоимость за встречу | < $0.50 | Cost estimator |
| Поддерживаемые форматы | 10+ аудио/видео | Тестирование |
| Конверсия onboarding | > 80% завершают настройку | Аналитика |

## 2. Пользовательские персоны

### Персона 1: IT-консультант (основная)

- **Профиль:** Проводит 5-10 обследований в неделю у заказчиков
- **Потребность:** Быстро получить протокол, требования и глоссарий после встречи
- **Боль:** Тратит вечера на написание протоколов, теряет детали
- **Ценность Aether:** Получает 6 артефактов за 5 минут вместо 3 часов ручной работы

### Персона 2: Руководитель проекта

- **Профиль:** Ежедневные стендапы, еженедельные статусы
- **Потребность:** Трекинг решений, задач и рисков по серии встреч
- **Боль:** Action items теряются, решения забываются
- **Ценность Aether:** Кумулятивный протокол по серии встреч, ничего не теряется

### Персона 3: Бизнес-аналитик

- **Профиль:** Сбор требований, GAP-анализ, формирование ТЗ
- **Потребность:** Структурированные требования с acceptance criteria
- **Боль:** Требования разбросаны по записям, нет единого реестра
- **Ценность Aether:** FR/NFR с приоритетами, бизнес-правила, глоссарий терминов

## 5. AI System Requirements

### 5.1 Промпты

6 специализированных промптов для извлечения артефактов:
- **System prompt:** Роль эксперта IT-аналитика с правилами: извлекать только явно сказанное, не додумывать, помечать неясности, сохранять терминологию заказчика
- **Temperature:** 0.1 для точных артефактов, 0.3 для стенограммы
- **Output:** Строгий JSON (не markdown) для программной обработки
- **Модификаторы типа встречи:** обследование, рабочая, демонстрация, приёмка

### 5.2 Обработка длинных записей

- Chunking: 60K токенов + 5K перекрытие
- Merge: объединение результатов чанков с дедупликацией
- Cost estimation: оценка стоимости до запуска обработки

### 5.3 Кумулятивный контекст

- При серии встреч по проекту: передача previous_artifacts для связности
- Отслеживание изменений: статусы new/changed/confirmed/contradicts

## 6. Нефункциональные требования

| Категория | Требование |
|-----------|-----------|
| **Производительность** | Запуск приложения < 2 сек, обработка 1 часа записи < 5 мин |
| **Безопасность** | API-ключи в Tauri Stronghold (зашифровано), HTTP-запросы через Rust backend (не видны в DevTools) |
| **Удобство** | Glassmorphism UI, 6 звуковых эффектов, анимации переходов, русский интерфейс |
| **Надёжность** | Graceful degradation при ошибках API, retry логика, предупреждения о качестве аудио |
| **Платформа** | Windows 10/11 x64 с WebView2 |
| **Размер** | Installer ~2 MB, RAM < 100 MB в простое |

## 7. Техническая архитектура

| Слой | Технология |
|------|-----------|
| Desktop runtime | Tauri 2.x (Rust) — NSIS installer |
| Frontend | React 19 + TypeScript 5.8 + Vite 7 |
| Стили | Tailwind CSS 4 + glassmorphism design system |
| Состояние | Zustand 5 (5 stores с persist middleware) |
| Анимации | Motion (ex-Framer Motion) |
| Звук | Howler.js + Web Audio API |
| AI: STT | OpenAI Whisper API |
| AI: LLM | Claude API (Sonnet 4.6) / OpenAI GPT-4 |
| Безопасность | Tauri Stronghold + reqwest (Rust HTTP) |
| Экспорт | docx (JS) + JSZip |

### Архитектура запросов

```
Frontend (React) → invoke() → Rust Backend → reqwest → API
                                                ↑
                              API-ключи в Stronghold (зашифрованы)
```

API-ключи никогда не покидают Rust — фронтенд передаёт данные через Tauri IPC, Rust добавляет авторизацию и отправляет запрос.

## 9. Риски и митигации

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Высокая стоимость API при больших файлах | Средняя | Высокое | Cost estimator перед обработкой, chunking, выбор модели |
| Низкое качество аудио → плохая транскрипция | Высокая | Высокое | Предупреждение о качестве, рекомендации по записи |
| CORS-блокировка в WebView | Средняя | Высокое | Все HTTP-запросы через Rust backend (reqwest) |
| Конфиденциальность данных клиентов | Средняя | Критическое | Локальная обработка, Stronghold, данные не покидают ПК |
| API rate limits | Низкая | Среднее | Rate limiter, очередь запросов, retry с backoff |
| Изменение API провайдеров | Низкая | Среднее | Абстракция llm.service, легко добавить нового провайдера |
| WebView2 недоступен на старых Windows | Низкая | Среднее | Автоматическая установка WebView2 через NSIS installer |
