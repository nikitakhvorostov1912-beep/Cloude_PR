# Финансы и закрытие периода: 1С:ERP УСО 2.5

## Казначейство

### Платёжный календарь
- **Путь:** Казначейство → Платёжный календарь
- **Назначение:** Основное рабочее место казначея — прогноз движения ДС по дням и счетам
- **Возможности:**
  - Прогноз остатков по банковским счетам и кассам на каждый день
  - Кассовые разрывы (отрицательные остатки) выделяются красным
  - Перепланирование платежей: перенос даты, разбивка суммы, смена счёта оплаты
  - Перевод заявок в статус "К оплате" одной командой
  - Детализация по счетам, валютам, подразделениям

### Лимиты расходования по статьям ДДС
- **Путь:** Казначейство → Лимиты расходования ДС (или через Бюджетирование)
- **Периодичность:** Обычно на месяц
- **Аналитика:** Статья ДДС + подразделение + организация
- **Типы лимитов:**
  - **Разрешающий** — при превышении блокирует проведение документа
  - **Дополнительно ограничивающий** — уточняет по доп. аналитикам (направление деятельности, партнёр)
  - **Информационный** — предупреждение без блокировки
- **Сверх лимита:** Признак в заявке "Сверх лимита" → дополнительное согласование

### Заявки на расходование ДС — процесс согласования

| Шаг | Статус | Ответственный | Действие |
|-----|--------|---------------|---------|
| 1 | Не согласована | Инициатор | Создание, статья ДДС, сумма, срок |
| 2 | Согласована | Руководитель подразделения | Проверка, лимит |
| 3 | К оплате | Казначей | Платёжный календарь, выбор счёта |
| 4 | Оплачена | Бухгалтер/казначей | Платёжное поручение |

## Бюджетирование по объектам строительства

- **Путь:** Бюджетирование → Бюджеты / Виды бюджетов / Монитор бюджетного процесса
- **Объект строительства** — ключевая аналитика бюджетов в УСО

**Виды бюджетов по объекту:**
- **БДР** — выручка от СМР, затраты на производство
- **БДДС** — плановые поступления от заказчика, плановые оплаты субподрядчикам и поставщикам
- **Инвестиционный бюджет** — капитальные вложения по объекту

**Связь смета → бюджет [требует проверки]:**
- Мастер обработки смет загружает сметную документацию (форматы ГрандСмета, Смета 3)
- Сопоставление видов работ и этапов из сметы с видами работ в системе
- Автоматическое формирование плановых показателей бюджета по объекту

**Контроль исполнения:**
- Рабочее место "Монитор бюджетного процесса" — визуальная оценка, отклонения
- План-фактный анализ в разрезе объекта, этапа, статьи затрат
- Лимиты: при расходе сверх бюджета — предупреждение или блокировка

## Закрытие месяца: полная последовательность

**Путь:** Финансовый результат и контроллинг → Закрытие месяца

| 7 | Списание расходов будущих периодов (РБП) | |
| 8 | Переоценка валютных средств | |
| 9 | Расчёт расходов по страхованию | |

## Незавершённое производство (НЗП) в строительстве

- НЗП = остаток затрат по объекту на сч.20, по которому ещё не подписан акт КС-2
- При закрытии месяца НЗП **не списывается автоматически** — остаётся на сч.20 до момента актирования
- При подписании КС-2 Внешней: затраты переносятся с НЗП на себестоимость реализованных работ
- Остаток НЗП переходит на следующий месяц и продолжает накапливаться

## Себестоимость строительства

### Счета производства

| Счёт | Назначение | Аналитика в строительстве |
|------|-----------|--------------------------|
| 20 | Основное производство (прямые затраты СМР) | Объект строительства, вид работ, этап, подразделение |
| 23 | Вспомогательные производства (механизмы, транспортный цех) | Вид производства, статья затрат |
| 25 | Общепроизводственные расходы (ИТР участка, амортизация техники) | Подразделение, статья затрат |
| 26 | Общехозяйственные расходы (управление, офис) | Организация, статья затрат |

### Схема формирования себестоимости объекта

```
Первичные документы месяца:
  ├── Передача материалов в производство → Дт 20 / Кт 10
  ├── Расчёт зарплаты рабочих            → Дт 20 / Кт 70
  ├── Акт субподрядчика                  → Дт 20 / Кт 60
  ├── Расходы механизмов (из сч.23)      → Дт 20 / Кт 23
  └── Прочие прямые затраты              → Дт 20

Регламентные операции закрытия:
  ├── Распределение сч.25 → Дт 20 / Кт 25 (по базе: ФОТ, прямые затраты и т.д.)
  ├── Распределение сч.26 → Дт 20 / Кт 26 (или директ-костинг → Дт 90.08 / Кт 26)
  └── Расчёт себестоимости — оборот Дт 20 = себестоимость объекта за месяц

При подписании КС-2 Внешней:
  └── Дт 90.02 / Кт 20 — списание себестоимости реализованных СМР
```

### Метод распределения сч.26 (настраивается в учётной политике)
- **"В себестоимость продукции"** — сч.26 распределяется на сч.20/23 → полная себестоимость объекта
- **"Директ-костинг"** — сч.26 закрывается на сч.90.08 "Управленческие расходы" → не входит в себестоимость объекта

## Управленческие отчёты в 1С ERP УСО 2.5

### Для прораба / ПТО

| Отчёт | Раздел меню | Назначение |
|-------|-------------|-----------|
| Диаграмма Ганта план-факт | Строительное производство → Отчёты | Контроль сроков |
| План-факт по реализации работ | Строительное производство → Отчёты | Контроль объёмов |
| Анализ потребления материалов | Строительное производство → Отчёты | Расход vs план |
| Журнал учёта выполненных работ (КС-6а) | Строительное производство → Отчёты | Нарастающий итог |

### Для финансового директора / казначея

| Отчёт | Раздел меню | Назначение |
|-------|-------------|-----------|
| Бюджет ДиР (план-факт) | Бюджетирование → Отчёты | Анализ доходов/расходов |
| Бюджет ДДС (план-факт) | Бюджетирование → Отчёты | Анализ денежных потоков |
| Платёжный календарь | Казначейство | Управление ДС |
| Себестоимость по объектам | Финансовый результат → Отчёты | Себестоимость объектов |
| Монитор бюджетного процесса | Бюджетирование → Монитор | Исполнение бюджетов |

### Для генерального директора

| Отчёт | Раздел меню | Назначение |
|-------|-------------|-----------|
| Управленческий баланс | Финансовый результат → Управленческие отчёты | Активы и пассивы |
| Отчёт о прибылях и убытках | Финансовый результат → Управленческие отчёты | П&У |
| Отчёт о движении ДС | Финансовый результат → Управленческие отчёты | ОДДС |
| Plan-фактный анализ договоров | Продажи → Анализ | Выполнение договоров |

**Примечание:** Точные пути в меню зависят от настройки ролевого интерфейса при конкретном внедрении [требует проверки].
