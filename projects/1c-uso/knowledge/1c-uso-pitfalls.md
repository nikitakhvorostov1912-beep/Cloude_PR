## P-001: НЕТ поля ВидДвижения у регистра ДоступностьВидовРабочихЦентров
**Дата:** 2026-03-05
**Область:** Разработка / Регистры
**Симптом:** Запрос к регистру накопления ДоступностьВидовРабочихЦентров падает с ошибкой "Поле не найдено"
**Причина:** Регистр является регистром остатков (не оборотным). Поле ВидДвижения существует только у оборотных регистров.
**Решение:** Убрать поле ВидДвижения из запроса. Использовать только поля из списка метаданных регистра (Измерения: ВидРабочегоЦентра, ДатаИнтервала; Ресурсы: Занято, ДоступностьПоВидуРЦ, ДоступностьПоРЦ; Реквизиты: МаксимальнаяДоступность и др.)
**Источник:** 1c-erp-development.md (проверено на реальной базе)

## P-002: Производственное подразделение не отмечено флагом "производит продукцию"
**Дата:** 2026-03-05
**Область:** НСИ / Структура предприятия
**Симптом:** При закрытии месяца затраты по СМР не распределяются корректно; себестоимость не формируется по объектам строительства
**Причина:** Подразделение, выполняющее СМР, не отмечено как производственное
**Решение:** НСИ и администрирование → Структура предприятия → открыть подразделение → установить флаг "Подразделение производит продукцию"

## P-003: Настройки учётной политики изменены ПОСЛЕ ввода начальных остатков
**Дата:** 2026-03-05
**Область:** НСИ / Учётная политика
**Симптом:** Документы проводятся по-старому, итоги в отчётах расходятся
**Причина:** Учётная политика применяется ретроспективно ко всем документам
**Решение:** Перепровести все документы после изменения учётной политики. Планировать изменения только на начало нового периода.

## P-004: Взаимозачёт аванса заказчика — двоение задолженности (62.01 и 62.02 открыты одновременно)
**Дата:** 2026-03-05
**Область:** Взаиморасчёты
**Симптом:** Акт выполненных работ проведён, но задолженность заказчика не закрывается; одновременно висит дебиторка по 62.01 и аванс по 62.02
**Причина:** Зачёт аванса не выполнен автоматически (способ зачёта в поступлении оплаты — "Не зачитывать") или детализация взаиморасчётов в договоре не совпадает
**Решение:** Казначейство → Взаимозачёты и списание задолженности → Взаимозачёт задолженности → выбрать контрагента → Рассчитать взаимозачёт. Проверить детализацию взаиморасчётов в договоре.

## P-005: НДС с аванса не восстановлен при зачёте
**Дата:** 2026-03-05
**Область:** Взаиморасчёты / НДС
**Симптом:** НДС задвоен в декларации: и с аванса (76.АВ → 68.02), и с реализации (90.03 → 68.02); к вычету НДС с аванса не принят
**Причина:** После проведения КС-2 Внешней не был запущен Помощник по учёту НДС для принятия к вычету НДС с аванса
**Решение:** Финансовый результат и контроллинг → НДС → Помощник по учёту НДС → "Принять НДС к вычету по авансам полученным" → проверить и провести документы

## P-006: Требование-накладная вместо строительных документов для списания МТР
**Дата:** 2026-03-05
**Область:** Строительство / Материалы
**Симптом:** Материалы списались, но не привязались к объекту строительства; управленческая аналитика по объекту не заполнена
**Причина:** Использован стандартный документ "Требование-накладная" вместо отраслевых документов УСО
**Решение:** Использовать "Расход материалов в строительстве" (управленческий) + "Передача материалов в производство" (бухгалтерский). Не использовать стандартную Требование-накладную для объектов строительства [требует подтверждения на конкретной инсталляции].

## P-007: КС-2 Внешняя проведена без КС-2 Внутренней
**Дата:** 2026-03-05
**Область:** Строительство / Документооборот
**Симптом:** Выручка в бухучёте есть, но данные управленческого учёта (непредъявленная выручка, аналитика по видам работ) некорректны
**Причина:** КС-2 Внешняя создана вручную без связи с КС-2 Внутренней
**Решение:** Соблюдать последовательность: Учёт выполненных работ (КС-6а) → КС-2 Внутренняя → КС-2 Внешняя (на основании КС-2 Внутренней)

## P-008: НЗП не списывается при закрытии месяца — думают, что ошибка
**Дата:** 2026-03-05
**Область:** Закрытие / Себестоимость
**Симптом:** Остаток по сч.20 не обнуляется при закрытии месяца
**Причина:** Это НОРМАЛЬНОЕ поведение для строительства! НЗП (незавершённое производство) остаётся на сч.20 до подписания КС-2 Внешней с заказчиком
**Решение:** Не паниковать. Проверить, что НЗП относится к объектам, по которым КС-2 ещё не подписана. Если объект актирован (КС-2 Внешняя проведена) — но НЗП всё равно висит, тогда проверить привязку документов к объекту.

## P-009: Детализация взаиморасчётов в договоре изменена после ввода документов
**Дата:** 2026-03-05
**Область:** Взаиморасчёты
**Симптом:** Документы проводятся с ошибками, взаиморасчёты "разъехались"
**Причина:** После изменения детализации в договоре ранее проведённые документы работают по-старому
**Решение:** Перепровести все документы по этому договору. Проводить изменения детализации только в начале нового периода или до ввода первого документа.

## P-010: Синхронные диалоги в асинхронном контексте внешней обработки
**Дата:** 2026-03-05
**Область:** Разработка / Внешние обработки (.epf)
**Симптом:** "Несоответствие типов (параметр номер '2')" при использовании ОписаниеОповещения
**Причина:** ОписаниеОповещения в контексте внешней обработки вызывает ошибку
**Решение:** Использовать СИНХРОННЫЕ методы: Вопрос(…), Предупреждение(…) — НЕ использовать ПоказатьВопрос(), ПоказатьПредупреждение() с ОписаниеОповещения
**Источник:** 1c-erp-development.md (проверено)

## P-011: Опция "Независимо вести партнёров и контрагентов" изменена на живой базе
**Дата:** 2026-03-05
**Область:** НСИ
**Симптом:** Ошибка "Данные ИНН и КПП уже указаны для другого контрагента" при редактировании; часть контрагентов недоступна для выбора в документах
**Причина:** При включённой опции к одному партнёру привязывалось несколько контрагентов. При отключении дублирующие элементы остаются физически в базе
**Решение:** Включать/выключать опцию ТОЛЬКО до начала работы с базой. На живой базе: копия → обработка "Поиск и удаление дублей" → выбрать основной элемент → объединить → удалить помеченные

## P-012: Статья расходов с вариантом "Распределять вручную" на производственных счетах
**Дата:** 2026-03-05
**Область:** Учётная политика / Закрытие месяца
**Симптом:** Затраты зависают на сч.20/25/26, не распределяются при закрытии месяца; нераспределённая сумма в аналитике по объекту
**Причина:** В статье расходов выбран вариант "Распределять вручную" + указан затратный счёт 20.01; программа не трогает эти суммы при закрытии месяца
**Решение:** Вариант "Вручную" применять только для статей типа сч.91.02 (недостачи, страхование). Для производственных расходов использовать "На себестоимость продукции (работ)"

## P-013: ГФУ (группа финансового учёта) номенклатуры изменена после оприходования
**Дата:** 2026-03-05
**Область:** НСИ / Закупки
**Симптом:** Новые поступления идут по новым счетам (н-р 10.06), старые партии остаются на прежних (10.01); при списании возможны отрицательные остатки; расхождения при закрытии месяца
**Причина:** ГФУ записывается в документ при выборе номенклатуры и фиксируется в партии. Изменение в карточке номенклатуры не пересчитывает оприходованные остатки
**Решение:** Настраивать ГФУ по виду номенклатуры ДО начала работы. При вынужденной смене: распровести все документы оприходования данной номенклатуры → изменить ГФУ → перепровести

## P-014: Передача материалов в производство с разных складов — ошибка "Разные отправители"
**Дата:** 2026-03-05
**Область:** Строительство / МТР
**Симптом:** При создании "Передача материалов в производство" с материалами из нескольких складов: "У выбранных распоряжений разные отправители"
**Причина:** Один документ передачи материалов может обслуживать только один склад-отправитель
**Решение:** Создавать отдельный документ для каждого склада-отправителя. Или: завести "склад-кладовую" при подразделении → перемещать туда материалы документом "Перемещение товаров" → передавать в производство уже с кладовой

## P-015: Ручная корректировка проводок в операционных документах ERP
**Дата:** 2026-03-05
**Область:** Регламентированный учёт / Антипаттерн
**Симптом:** После ручной корректировки и следующего перепроведения документа (н-р при закрытии месяца) правки сбрасываются; регламентный и оперативный учёт расходятся
**Причина:** Флаг "Ручная корректировка" отключает автоматическое формирование движений. Пакетное перепроведение игнорирует документ в части проводок
**Решение:** НИКОГДА не использовать ручную корректировку в операционных документах. Для исправлений использовать отдельный документ "Операция (регламентированный учёт)"

## P-016: Незакрытые авансы покупателя из-за несовпадения "Направления деятельности"
**Дата:** 2026-03-05
**Область:** Взаиморасчёты
**Симптом:** Аванс от заказчика не зачитывается при выставлении КС-3; в акте сверки одновременно висит и наш долг по возврату аванса, и долг заказчика за работы
**Причина:** Включено ведение "Направлений деятельности". Платёжный документ — с одним направлением деятельности, документ реализации — с другим. Программа считает их разными объектами аналитики
**Решение:** При использовании направлений деятельности — обеспечить единообразное заполнение в цепочке "Платёж → Договор → Документ реализации". При несовпадении: Взаимозачёт задолженности → "Подобрать по остаткам"

## P-017: Расхождение регистров при возврате товара с другого склада (баг версии)
**Дата:** 2026-03-05
**Область:** Закрытие месяца
**Симптом:** Закрытие месяца: "Обнаружены расхождения по количеству между регистрами 'Выручка и себестоимость продаж' и 'Себестоимость товаров'". Операция заблокирована.
**Причина:** В "Возврат товаров от клиента" указан склад, отличный от склада в исходной реализации. Ошибка поведения конфигурации (зафиксирована до версии 2.5.12.270)
**Решение:** Обновить конфигурацию до 2.5.12.270+. Временный обходной путь: настройки закрытия месяца → данная проверка → изменить важность с "Ошибка" на "Информация"

## P-018: Дубли НСИ при переносе данных из старой системы
**Дата:** 2026-03-05
**Область:** НСИ / Внедрение
**Симптом:** Десятки позиций "Кирпич" с разными названиями в справочнике; аналитика по МТР бесполезна; менеджеры создают новые элементы вместо поиска
**Причина:** Данные перенесены без нормализации; регламент НСИ не разработан
**Решение:** Нормализацию НСИ выполнять ДО переноса. Разработать регламент: классификатор, единицы измерения, правила именования. Назначить ответственного за НСИ. После переноса: обработка "Поиск и удаление дублей". Для строительства особо проверить справочник "Объекты строительства"
