# Взаиморасчёты: 1С:ERP УСО 2.5

## Схема расчётов с заказчиком строительства

```
1. Заказчик → аванс
   [Поступление на расчётный счёт] вид "Оплата от покупателя"
   Дт 51 / Кт 62.02 — аванс полученный
        ↓
2. Начисление НДС с аванса (в течение 5 рабочих дней)
   [Счёт-фактура выданный (аванс)] — через Помощник НДС
   Дт 76.АВ / Кт 68.02
        ↓
3. Выполнение работ и внутренняя приёмка
   [КС-2 Внутренняя]
        ↓
4. Сдача работ заказчику (КС-2/КС-3)
   [Реализация СМР — КС-2 Внешняя] + [Счёт-фактура выданный]
   Дт 62.01 / Кт 90.01 (выручка); Дт 90.03 / Кт 68.02 (НДС)
        ↓
5. Зачёт аванса
   [Взаимозачёт задолженности] или автоматически
   Дт 62.02 / Кт 62.01 — зачёт аванса
   + Восстановление НДС: Дт 68.02 / Кт 76.АВ
        ↓
6. Заказчик → оплата остатка
   [Поступление на расчётный счёт] вид "Оплата от покупателя"
```

## Документы расчётов с заказчиком

### Поступление на расчётный счёт (аванс / оплата)
- **Путь:** Казначейство → Банковские выписки → Поступление на расчётный счёт
- **Вид операции:** "Оплата от покупателя"
- **Ключевые реквизиты:** Контрагент (заказчик), Договор, Сумма, Статья ДДС
- **Важный реквизит:** Способ зачёта аванса — "Автоматически" / "По документу" / "Не зачитывать"
  - "Автоматически" — FIFO-зачёт при следующей реализации
  - "Не зачитывать" — аванс остаётся на сч.62.02 для ручного зачёта

### Счёт-фактура выданный (аванс)
- **Путь:** Финансовый результат → НДС → Помощник по учёту НДС → "Выставить счета-фактуры по полученным авансам"
- **Назначение:** Обязательный документ при получении предоплаты, НДС с аванса
- **Срок:** В течение 5 календарных дней с получения аванса (ст.168 НК РФ)
- **Проводка:** Дт 76.АВ / Кт 68.02
- **Результат:** Запись в Книгу продаж

### Взаимозачёт задолженности (зачёт аванса)
- **Путь:** Казначейство → Взаимозачёты и списание задолженности → Взаимозачёт задолженности
- **Или:** В документе реализации → кнопка "Зачесть/перенести аванс"
- **Виды операций:** "Перенос аванса клиента" / "По встречным обязательствам" / "Произвольный взаимозачёт"
- **Ключевые реквизиты:** Организация, Контрагент, Договор, Документ-основание (КС-2 Внешняя)
- **Кнопка "Рассчитать взаимозачёт"** — автоматически определяет сумму
- **Проводка:** Дт 62.02 / Кт 62.01

### Восстановление (вычет) НДС с аванса при зачёте
- **Когда:** В периоде зачёта аванса (после проведения КС-2 Внешней)
- **Путь:** Финансовый результат → НДС → Помощник по учёту НДС → "Принять НДС к вычету по авансам полученным"
- **Проводка:** Дт 68.02 / Кт 76.АВ
- **Результат:** Запись в Книгу покупок; уменьшение НДС к уплате в бюджет

## Схема расчётов с субподрядчиком

```
1. Аванс субподрядчику
   [Списание с расчётного счёта] вид "Оплата поставщику"
   Дт 60.02 / Кт 51
        ↓
2. Субподрядчик выполняет работы → предъявляет КС-2
   [Акт о приёмке выполненных работ субподрядчика]
   Дт 20 / Кт 60.01 (стоимость работ); Дт 19 / Кт 60.01 (входящий НДС)
        ↓
3. Счёт-фактура от субподрядчика
   [Счёт-фактура полученный]
   НДС к вычету: Дт 68.02 / Кт 19
   Запись в Книгу покупок
        ↓
4. Зачёт аванса субподрядчику
   [Взаимозачёт задолженности] — вид "Перенос аванса поставщику"
   Дт 60.01 / Кт 60.02
        ↓
5. Доплата субподрядчику (остаток)
   [Списание с расчётного счёта]
   Дт 60.01 / Кт 51
```

**Объекты расчётов с субподрядчиком** (настраивается в договоре):
- По договорам / По заказам поставщику / По расчётным документам

**Типичная проблема:** У контрагента одновременно есть "Долг партнёра" и "Наш долг" → незавершённый зачёт. Решение: "Взаимозачёт задолженности" в Казначействе.

## Акт сверки взаиморасчётов

- **Путь:**
  - С заказчиком: Продажи → Сверки взаиморасчётов → Акт сверки
  - С субподрядчиком: Закупки → Сверки взаиморасчётов → Акт сверки
  - Универсально: Казначейство → Сверки взаиморасчётов
- **Способ создания:** Через "Помощник формирования акта сверки" (рекомендуется) или вручную
- **Ключевые реквизиты:** Период, Организация, Контрагент, Тип (покупатель/поставщик/прочие), Договор (если пусто — по всем договорам), Дата сверки
- **Заполнение:** Кнопка "Заполнить" — автоматически подбирает все движения за период
- **Содержание:** Колонка организации (отгрузки, поступления оплат), Колонка контрагента, Итоговое сальдо
- **Статус:** "Сверено без разногласий" / "Сверено с разногласиями" / "Не подписан"
- **После формирования:** Отправка через ЭДО или электронную почту; двусторонняя печатная форма

## Ключевые настройки взаиморасчётов

**Детализация взаиморасчётов** (настраивается в договоре с контрагентом):
- По договорам
- По заказам поставщику / по заказам покупателя
- По расчётным документам (накладным)
- Аванс по договорам, долг по накладным

**ВАЖНО:** При изменении детализации после проведения документов — необходимо перепровести все связанные документы!
