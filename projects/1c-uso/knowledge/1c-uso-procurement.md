# Закупки и снабжение МТР: 1С:ERP УСО 2.5

## Схема процесса (от потребности до оплаты)

```
[Потребность в МТР из плана выполнения работ / вручную]
        ↓
[Заказ на внутреннее потребление / Внутренний заказ]
        ↓ (через АРМ "Формирование заказов по потребностям")
[Заказ поставщику]
        ↓
[Приобретение товаров и услуг] — поступление МТР на склад
        ↓
[Счёт-фактура полученный] — НДС к вычету
        ↓
[Заявка на расходование ДС] — инициирование оплаты
        ↓ (согласование)
[Списание безналичных ДС] — фактическое платёжное поручение
```

## Документы закупок

### Заказ на внутреннее потребление (Внутренний заказ)
- **Путь:** Склад и доставка → Внутренние заказы → Заказы на внутреннее потребление
- **Назначение:** Фиксирует потребность подразделения/объекта строительства в МТР
- **Ключевые реквизиты:** Организация, Подразделение/Объект строительства [требует проверки], Номенклатура + количество, Склад назначения, Дата желаемого поступления, Статья затрат
- **Источники создания:** Вручную или автоматически из плана выполнения работ (в УСО)
- **Результат:** Формируется потребность → видна в АРМ "Обеспечение потребностей"

### Заказ поставщику
- **Путь:** Закупки → Заказы поставщикам → Создать → Новый
- **Назначение:** Центральный документ управления закупками — фиксирует договорённость с поставщиком
- **Ключевые реквизиты:** Поставщик/Контрагент, Соглашение (цены, условия оплаты), Договор, Организация, Склад доставки, Номенклатура/кол-во/цена/НДС, График поставок, График платежей (аванс/постоплата)
- **Статусы:** Не согласован → Согласован → Подтверждён → Закрыт
- **Создаётся:** Вручную или через АРМ "Формирование заказов по потребностям"
- **Результат:** Резервирование, ожидаемые поступления на складе, контроль исполнения

### Соглашение об условиях закупок
- **Путь:** Закупки → НСИ и настройки → Соглашения с поставщиками
- **Назначение:** Справочник условий поставщика — автоматически подтягивается в заказ
- **Реквизиты:** Партнёр, Валюта, Порядок оплаты (% аванса), Условия поставки, Прайс-лист

### Приобретение товаров и услуг (поступление МТР)
- **Путь:** Закупки → Документы закупки (все) → Создать → Приобретение товаров и услуг
- **Или:** На основании Заказа поставщику → кнопка "Создать на основании"
- **Назначение:** Первичный документ поступления МТР от поставщика на склад (аналог ТОРГ-12)
- **Ключевые реквизиты:**
  - Основное: Поставщик, Соглашение, Организация, Договор, Склад, Заказ поставщику (привязка)
  - Товары: Номенклатура, Количество факт/по документу, Цена, % НДС, Сумма
  - **Аналитика расходов (объект строительства)** — ключевой реквизит для стройки [требует проверки точного поля в конфигурации]
  - Статья расходов (для услуг и работ)
- **Результат:** Приход МТР на склад, задолженность перед поставщиком (сч.60), запись по сч.10

### Счёт-фактура полученный
- **Путь:** Из документа поступления → кнопка "Зарегистрировать счёт-фактуру"
- **Ключевые реквизиты:** Номер и дата от поставщика, привязка к поступлению, сумма НДС
- **Результат:** Запись в книгу покупок, право на вычет НДС

### Акт о расхождениях (при несоответствии факта и документов)
- **Путь:** На основании Приобретения → Создать на основании → Акт о расхождениях
- **Реквизиты:** Количество по документу, Количество факт, Количество расхождений
- **Статусы:** Не согласовано → Согласовано
- **Решения по позициям:** Оформить излишки / Вернуть / На прочие доходы / Оформить допоставку

### Возврат товаров поставщику
- **Путь:** Закупки → Документы закупки → Возврат товаров поставщику
- **Или:** На основании Акта о расхождениях
- **Результат:** Уменьшение складских остатков и задолженности перед поставщиком

### Заявка на расходование денежных средств
- **Путь:** Казначейство → Заявки на расходование ДС → Оплата поставщику
- **Назначение:** Инициация платежа, проходит согласование, плановый документ в платёжном календаре
- **Ключевые реквизиты:** Организация, Подразделение, Операция "Оплата поставщику", Получатель+счёт, Сумма, Дата платежа; Расшифровка: Объект расчётов (Заказ поставщику/Договор), **Статья ДДС** (обязательна для управленческого учёта), НДС
- **Статусы:** Не согласовано → На согласовании → Согласовано → К оплате → Оплачено
- **Результат:** Включение в платёжный календарь, блокировка суммы в плане ДДС

### Списание безналичных ДС (Платёжное поручение исходящее)
- **Путь:** Казначейство → Безналичные платежи → Списание безналичных ДС
- **Или:** На основании Заявки на расходование ДС
- **Назначение:** Фактическое отражение оплаты поставщику через банк
- **Ключевые реквизиты:** Счёт организации, Операция "Оплата поставщику", Сумма, Номер п/п, Получатель; флаг "Проверено банком" — после загрузки выписки
- **Результат:** Уменьшение остатков на банковском счёте, закрытие задолженности перед поставщиком (сч.60)

## Особенности привязки МТР к объекту строительства

**Механизм 1 — Аналитика расходов в поступлении:**
В "Приобретении товаров и услуг" для услуг/работ указывается: Статья расходов + Аналитика расходов (Объект строительства) [требует проверки точного реквизита в конфигураторе]

**Механизм 2 — Заказ под потребность объекта:**
При создании Заказа поставщику из потребности объекта строительства — связь "заказ↔объект" сохраняется автоматически

**Механизм 3 — Склад объекта:**
Создать отдельный склад "Склад на объекте ЖК Сосновый" — все поступления на него автоматически относятся к объекту

**Механизм 4 — АРМ обеспечения потребностей:**
Служба МТО видит потребности по объектам, оценивает складские остатки и формирует Заказы поставщикам

## Давальческие материалы субподрядчику

```
Схема передачи:
[Материалы на складе генподрядчика]
        ↓
[Заказ переработчику]
  Производство → Передача в переработку → Заказы переработчикам
        ↓
[Передача сырья переработчику]
  Формируется автоматически из Заказа переработчику
  Бухучёт: Дт 10.07 / Кт 10.01
        ↓
[Субподрядчик выполняет работы]
        ↓
[Поступление из переработки / Отчёт переработчика]
  Фиксирует: выполненные работы, потреблённые материалы, остатки к возврату
        ↓
[Возврат неиспользованных материалов] (при наличии)
```

**ВАЖНО:** Давальческие материалы остаются на балансе генподрядчика (сч.10.07). Субподрядчик не вправе передавать их третьим лицам.
