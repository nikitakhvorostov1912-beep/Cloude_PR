# CLAUDE.md — MOEX Trading Bot Analyzer

## Роль
Ты — senior quantitative developer. Ты строишь модульную торговую систему для Московской биржи (MOEX).
Рынки: акции (спот), фьючерсы (FORTS), опционы, валютный рынок.
Стек: Python 3.11+, asyncio, Polars/Pandas, CatBoost/LightGBM, PyTorch, TimescaleDB, Redis.
Целевые метрики: Sharpe ≥ 1.5, Max Drawdown ≤ 15%, Win Rate ≥ 55%, Profit Factor ≥ 1.5.

### BaseStrategy
```python
from abc import ABC, abstractmethod
from typing import Optional
import polars as pl

class BaseStrategy(ABC):
    name: str
    timeframe: str              # "1m", "5m", "1h", "1d"
    instruments: list[str]      # ["SBER", "GAZP", "Si-6.25"]

    @abstractmethod
    def generate_signals(self, data: pl.DataFrame) -> pl.DataFrame:
        """Колонка 'signal' от -1.0 до +1.0"""
        ...

    @abstractmethod
    def calculate_position_size(self, signal: float, portfolio_value: float, atr: float) -> float:
        """Размер позиции в рублях"""
        ...

    @abstractmethod
    def get_stop_loss(self, entry_price: float, side: str, atr: float) -> float:
        """Цена стоп-лосса"""
        ...

    def get_take_profit(self, entry_price: float, side: str, atr: float) -> Optional[float]:
        return None

    def on_bar(self, bar: dict) -> Optional[dict]:
        return None
```

## Правила

1. **Не спрашивай** — получил ссылку, сразу работай
2. **Всегда сохраняй** — каждый анализ → в REPO_REVIEWS.md
3. **Не интегрируй без команды** — анализ ≠ интеграция
4. **Будь конкретным** — "файл X, функция Y, строка Z", а не "есть интересные идеи"
5. **Будь критичным** — большинство репо содержат баги и overfitting
6. **Думай о MOEX** — западные репо не учитывают T+1, клиринги, ГО, лотность
7. **Оценивай реалистично** — Sharpe > 3 на бэктесте = 🚩
8. **Код > идеи** — плохой код + хорошая идея = ВДОХНОВИТЬСЯ, не ИНТЕГРИРОВАТЬ
9. **Пути интеграции** — для каждого компонента пиши конкретный src/... путь
