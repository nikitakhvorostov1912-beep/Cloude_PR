# Internal Comms — Внутренние коммуникации

Помогает создавать структурированные внутренние документы: статус-отчёты, обновления для руководства, рассылки, FAQ, инцидент-репорты.

> Источник: https://github.com/anthropics/skills (Apache-2.0)

## Когда использовать

- Статус-отчёт по проекту
- 3P обновления (Progress, Plans, Problems)
- Рассылка/newsletter команде
- FAQ по продукту или процессу
- Инцидент-репорт
- Обновление для руководства

## Когда НЕ использовать

- Внешние коммуникации (маркетинг, PR) → используй маркетинговые скиллы
- Техническая документация → используй `doc-updater`
- Презентация → используй `pptx-generation`

## Типы документов

### 1. 3P Update (Progress, Plans, Problems)
Краткое обновление для команды, читается за 30-60 секунд.

```markdown
## [Команда/Проект] — 3P Update
📅 Период: [даты]

### ✅ Progress (Прогресс)
- [1-3 data-driven предложения о завершённом]

### 📋 Plans (Планы)
- [1-3 конкретных следующих шага с дедлайнами]

### ⚠️ Problems (Проблемы)
- [0-2 блокера или риска с предложенным решением]
```

### 2. Статус-отчёт
```markdown
## Статус-отчёт: [Проект]
📅 [Дата] | Автор: [имя]

### Общий статус: 🟢 / 🟡 / 🔴

### Ключевые метрики
| Метрика | Текущее | Цель | Тренд |
|---------|---------|------|-------|
| [метрика] | [значение] | [цель] | ↑/↓/→ |

### Достижения за период
1. [достижение + метрика]

### Следующие шаги
1. [шаг + ответственный + дедлайн]

### Риски и блокеры
- [риск]: [митигация]
```

### 3. Company Newsletter
```markdown
## [Компания] Newsletter — [Месяц/Квартал]

### 🎉 Главное
[1-2 ключевых события]

### 📊 Метрики
[ключевые цифры за период]

### 👥 Команда
[новые сотрудники, достижения, благодарности]

### 📅 Предстоящее
[события, дедлайны]
```

### 4. FAQ
```markdown
## FAQ: [Тема]

### Общие вопросы

**В: [Вопрос]?**
О: [Краткий, чёткий ответ. 2-3 предложения максимум.]

**В: [Вопрос]?**
О: [Ответ с ссылкой на детали если нужно.]
```

### 5. Инцидент-репорт
```markdown
## Инцидент-репорт: [Название]

### Сводка
- **Severity**: P1/P2/P3/P4
- **Время обнаружения**: [datetime]
- **Время решения**: [datetime]
- **Влияние**: [кто затронут, масштаб]

### Timeline
- [HH:MM] — [событие]
- [HH:MM] — [действие]

### Корневая причина
[Описание]

### Исправление
[Что было сделано]

### Предотвращение
1. [мера + ответственный + дедлайн]
```

## Принципы

- **Краткость**: читатель должен понять суть за 60 секунд
- **Data-driven**: цифры и метрики вместо общих слов
- **Actionable**: каждый пункт → конкретное действие
- **Структура**: единый формат для узнаваемости
- **Тон**: профессиональный, дружелюбный, без канцелярита
