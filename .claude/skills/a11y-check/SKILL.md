# /a11y-check — Проверка доступности (Accessibility)

## Когда использовать
После изменений в UI компонентах, перед релизом.

## Инструкции

### 1. Семантический HTML
- Заголовки идут по порядку (h1 → h2 → h3, без пропусков)?
- Используются семантические теги (`nav`, `main`, `aside`, `section`, `article`)?
- Нет `<div>` с onClick вместо `<button>`?
- `<a>` имеет href, `<button>` имеет type?

### 2. ARIA атрибуты
- Интерактивные элементы имеют `aria-label` или видимый текст?
- Иконки-кнопки имеют `aria-label`?
- Модальные окна имеют `aria-modal`, `role="dialog"`?
- Загрузка: `aria-busy`, `aria-live` для динамического контента?
- Табы: `role="tablist"`, `role="tab"`, `aria-selected`?

### 3. Клавиатурная навигация
- Все интерактивные элементы доступны через Tab?
- Порядок фокуса логичный?
- Есть видимый focus indicator (outline)?
- Escape закрывает модальные окна?

### 4. Контраст и визуал
- Текст имеет контраст ≥ 4.5:1 (AA)?
- Крупный текст ≥ 3:1?
- Не только цвет передаёт информацию (добавлены иконки/текст)?
- Тёмная тема: проверь контраст отдельно

### 5. Формы
- Все input имеют `<label>` (или `aria-label`)?
- Ошибки валидации связаны с полем через `aria-describedby`?
- Required поля обозначены (`aria-required` или визуально)?

### 6. Медиа
- Изображения имеют `alt` текст?
- Декоративные изображения: `alt=""`?

## Формат отчёта

```
## ♿ Проверка доступности (WCAG 2.1 AA)

### Семантика: {OK / Проблемы}
### ARIA: {OK / Проблемы}
### Клавиатура: {OK / Проблемы}
### Контраст: {OK / Проблемы}
### Формы: {OK / Проблемы}

### Проблемы
{файл:строка — описание}

### Вердикт: PASS / FAIL
```
