# TO-BE Optimization — Оптимизация процессов под 1С

Создай оптимизированные TO-BE процессы на основе AS-IS и GAP-анализа.

## Методика оптимизации

### Принципы
1. **Автоматизация** — заменить ручные операции типовым функционалом 1С
2. **Устранение дублирования** — один ввод данных, повторное использование
3. **Стандартизация** — привести к типовым документам и справочникам 1С
4. **Контроль** — добавить автоматические проверки и уведомления
5. **Прозрачность** — обеспечить отслеживание статусов и отчётность

### Типичные оптимизации

| AS-IS проблема | TO-BE решение в 1С |
|---------------|-------------------|
| Ручной ввод в Excel | Документ в 1С с автозаполнением |
| Проверка остатков в двух системах | Единый регистр остатков в 1С |
| Согласование по email | Бизнес-процесс согласования в 1С |
| Отчёт собирается вручную | Типовой/доработанный отчёт |
| Нет контроля сроков | Задания + напоминания в 1С |
| Дублирование документов | Цепочка документов с проведением |
| Ручной расчёт цен | Типы цен + скидки в 1С |

## Правила
- Каждый TO-BE шаг привязан к документу/отчёту 1С
- Указать уровень автоматизации: Автоматизировано / Частично / Ручное
- Improvements привязаны к pain_points из AS-IS
- Формировать BPMN JSON для TO-BE (для генерации Visio)
