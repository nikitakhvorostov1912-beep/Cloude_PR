# Code Review — Полный обзор кода

Проведи полный code review указанных файлов или всего проекта.

## Области проверки

### 1. Архитектура
- Соответствие паттернам проекта (FastAPI + Next.js)
- Разделение ответственности (services / routes / models)
- Корректные зависимости между модулями
- Нет циклических импортов

### 2. Безопасность
- Нет хардкоженных секретов (API ключи, пароли)
- Валидация входных данных (Pydantic на backend, zod/TS на frontend)
- Нет SQL/command injection уязвимостей
- CORS настроен корректно
- Путь к файлам валидируется (нет path traversal)

### 3. Производительность
- Нет N+1 запросов
- Большие файлы обрабатываются потоково
- Правильное использование async/await
- Нет блокирующих операций в async контексте
- React: нет лишних ре-рендеров, мемоизация где нужно

### 4. Качество кода
- Type hints на Python, TypeScript типизация
- Обработка ошибок (try/except с конкретными исключениями)
- Логирование ключевых операций
- Нет мёртвого кода, неиспользуемых импортов
- Имена переменных/функций отражают назначение
- Пользовательские сообщения на русском языке

### 5. Тестируемость
- Код можно протестировать (dependency injection)
- Нет глобального состояния
- Функции делают одну вещь

## Формат отчёта

Для каждой проблемы:
```
[СЕРЬЁЗНОСТЬ] файл:строка — Описание
  Было: <код>
  Стало: <исправленный код>
```

Серьёзности: CRITICAL, HIGH, MEDIUM, LOW, INFO

В конце:
- Общая оценка: A/B/C/D/F
- Топ-3 приоритетных исправления
- Что сделано хорошо (обязательно отметь)

После ревью — ИСПРАВЬ все CRITICAL и HIGH проблемы.
