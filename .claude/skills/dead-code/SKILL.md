# /dead-code — Поиск мёртвого кода

## Когда использовать
При рефакторинге, перед релизом, для уменьшения размера проекта.

## Инструкции

### 1. Python (Backend)
1. Найди все определения функций/классов (`def `, `class `)
2. Для каждого проверь — есть ли импорт или вызов в другом файле?
3. Проверь:
   - Неиспользуемые импорты (import без использования)
   - Переменные, которые присваиваются но не читаются
   - Функции с `pass` или `...` body (заглушки)
   - Закомментированный код (больше 3 строк)
   - `# TODO` и `# FIXME` — их не должно быть в production

### 2. TypeScript (Frontend)
1. Найди все `export` в `src/`
2. Для каждого экспорта проверь — импортируется ли где-то?
3. Проверь:
   - Неиспользуемые компоненты (файл есть, импорта нет)
   - Неиспользуемые типы/интерфейсы
   - `console.log`, `console.warn` — удалить
   - Закомментированный код

### 3. Orphan файлы
- Файлы, которые никуда не импортируются и не являются entry points
- Пустые `__init__.py` без re-exports
- Неиспользуемые конфиги

### 4. Unreachable код
- Код после `return`, `break`, `continue`
- `if False:` блоки
- Except блоки, которые никогда не срабатывают
