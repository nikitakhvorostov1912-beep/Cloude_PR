# Skill Seekers — Конвертер документации в скиллы

Автоматическое создание SKILL.md файлов из сайтов документации, GitHub репозиториев и PDF.

> Источник: https://github.com/yusufkaraaslan/Skill_Seekers

## Когда использовать

- Нашёл полезную документацию и хочешь сделать из неё скилл
- Нужен скилл по конкретной библиотеке/фреймворку
- Конвертация README.md проекта в скилл
- Создание скилла из нескольких связанных страниц документации

## Когда НЕ использовать

- Создание скилла с нуля → используй `/anthropics-skill-development`
- Простой README → просто прочитай его через Read
- Нужна только одна страница → используй WebFetch

## Workflow

### 1. Определи источник

| Тип | Пример |
|-----|--------|
| Сайт документации | https://docs.example.com |
| GitHub репо | https://github.com/user/repo |
| PDF файл | /path/to/documentation.pdf |
| Несколько URL | Список страниц документации |

### 2. Извлеки контент

```bash
# Из одного URL
WebFetch URL → извлечь структурированный контент

# Из GitHub репо
gh api repos/{owner}/{repo}/contents → найти README и ключевые файлы
WebFetch raw.githubusercontent.com → извлечь содержимое

# Из нескольких URL
Для каждого URL → WebFetch → объединить контент
```

### 3. Структурируй в SKILL.md

```markdown
---
name: [kebab-case-name]
description: "[Что делает скилл. Когда использовать. 1-2 предложения.]"
allowed-tools: [список инструментов]
---

# [Название] — [Краткое описание]

[Описание на 2-3 предложения]

> Источник: [URL]

## Когда использовать
- [триггер 1]
- [триггер 2]

## Когда НЕ использовать
- [ситуация] → [альтернатива]

## [Основные секции из документации]

## Примеры использования

## Обработка ошибок
```

### 4. Валидация

Проверь готовый SKILL.md:
- [ ] Frontmatter корректный (name, description)
- [ ] Описание содержит триггерные слова
- [ ] Секции "Когда / Когда НЕ" присутствуют
- [ ] Нет копирования текста 1:1 (перефразировано)
- [ ] Примеры кода адаптированы к проекту
- [ ] Русский язык в описании

### 5. Установка

```bash
mkdir -p .claude/skills/[name]/
# Сохрани SKILL.md в папку
```

## Шаблон SKILL.md

```markdown
---
name: example-skill
description: "Описание: что, когда, зачем. Используй когда нужно [триггер]."
allowed-tools: Bash,Read,Write
---

# Example Skill — Описание

Краткое описание на 2-3 предложения.

> Источник: [URL источника]

## Когда использовать

- Триггер 1
- Триггер 2
- Триггер 3

## Когда НЕ использовать

- [Ситуация] → используй [альтернативу]

## Основной контент

[Структурированные инструкции]

## Примеры

[Код с пояснениями]

## Обработка ошибок

- [Ошибка]: [решение]
```

## AI-улучшения

При конвертации автоматически:
1. **Conflict detection** — проверяет нет ли конфликтов с существующими скиллами
2. **OCR cleanup** — чистка артефактов из PDF
3. **Structure optimization** — переструктурирование для лучшего retrieval
4. **Deduplication** — удаление дублирующей информации
