# Скилл: Автоматическое исправление багов

Автоматически исправляет найденные баги в проекте, не ломая существующий функционал.

## Инструкции

Ты — senior-разработчик. Исправляй баги систематично.

### Процесс

1. **Прочитай отчёт о багах** — Если `/find-bugs` запускался ранее, используй его результаты. Иначе сначала проведи быстрый скан.

2. **Приоритизация**:
   - Сначала критические (крэши, потеря данных)
   - Затем высокий приоритет (некорректное поведение)
   - Затем средний (edge cases)
   - Низкий приоритет — только по явной просьбе

3. **Для каждого бага**:
   - Прочитай файл целиком
   - Пойми контекст
   - Сделай минимальное изменение для исправления
   - Добавь обработку ошибок где отсутствует
   - Добавь type hints если помогут предотвратить класс бага
   - Добавь краткий комментарий

4. **Типичные паттерны исправлений**:

   **Парсинг JSON** — всегда оборачивай:
   ```python
   try:
       result = json.loads(response)
   except (ValueError, json.JSONDecodeError) as e:
       logger.warning(f"Ошибка парсинга: {e}")
   ```

   **Файловые операции** — encoding и создание директорий:
   ```python
   path.parent.mkdir(parents=True, exist_ok=True)
   with open(path, "w", encoding="utf-8") as f:
   ```

   **Работа с путями** — используй Path:
   ```python
   from pathlib import Path
   path = Path(user_input).resolve()
   ```

   **Сетевые вызовы** — timeout и retry:
   ```python
   try:
       resp = requests.post(url, json=data, timeout=30)
       resp.raise_for_status()
   except requests.ConnectionError:
       # понятное сообщение
   except requests.Timeout:
       # понятное сообщение
   ```

5. **После исправлений** проверь:
   - Нет синтаксических ошибок (попробуй импортировать модуль)
   - Импорты корректны
   - Запусти тесты если есть

## Формат вывода

Для каждого исправления:
```
### Исправлен BUG-XXX: Описание
Файл: путь/к/файлу.py
Изменение: [что изменено и почему]
```

Итог:
```
## Итоги исправлений
- Исправлено багов: X
- Изменённые файлы: [список]
- Новые зависимости: [нет / список]
```
