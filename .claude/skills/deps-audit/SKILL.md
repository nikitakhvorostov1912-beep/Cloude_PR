# /deps-audit — Аудит зависимостей

## Когда использовать
Еженедельно, перед релизом, после добавления новых пакетов.

## Инструкции

### 1. Python (requirements.txt)
1. Прочитай `projects/survey-automation/backend/requirements.txt`
2. Для каждого пакета проверь:
   - Есть ли более новая версия? (поищи в интернете `{package} latest version`)
   - Есть ли известные CVE? (поищи `{package} CVE vulnerability`)
   - Pinned ли версия (==)? Должна быть pinned
3. Запусти `pip audit` если установлен
4. Проверь совместимость версий между собой

### 2. Node.js (package.json)
1. Прочитай `projects/survey-automation/frontend/package.json`
2. Проверь:
   - Устаревшие пакеты (major версии позади)
   - Deprecated пакеты
   - Известные уязвимости
3. Запусти `npm audit` если возможно

### 3. Лицензии
- Проверь что нет GPL пакетов в коммерческом проекте (если применимо)
- Допустимые: MIT, Apache-2.0, BSD, ISC

### 4. Supply chain
- Нет ли подозрительных пакетов (typosquatting)?
- Все пакеты из официальных репозиториев (PyPI, npm)?

## Формат отчёта

```
## 📦 Аудит зависимостей

### Python ({N} пакетов)
| Пакет | Текущая | Последняя | CVE | Статус |
|-------|---------|-----------|-----|--------|

### Node.js ({N} пакетов)
| Пакет | Текущая | Последняя | CVE | Статус |
|-------|---------|-----------|-----|--------|

### Критические уязвимости: {N}
### Устаревшие пакеты: {N}

### Вердикт: PASS / FAIL
```
