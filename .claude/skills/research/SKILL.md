# /research — Глубокое исследование

Запускает агент deep-researcher для многофазного исследования темы.
Claude ЖДЁТ завершения агента и НЕ отвечает до получения отчёта.

## Использование

```
/research [тема]
/research --quick [тема]
```

## Инструкции для Claude

Когда пользователь вызывает `/research`, выполни следующие шаги:

### 1. Парсинг команды

Извлеки из сообщения пользователя:
- **Тема** — что исследовать
- **Режим** — полный (по умолчанию) или `--quick` (быстрый)
- **Контекст** — если тема связана с текущим проектом

Примеры:
- `/research Electron vs Tauri` → тема: "Electron vs Tauri для десктоп-приложения"
- `/research --quick лучший ORM для Python` → быстрый режим, тема: "ORM для Python"
- `/research как реализовать аутентификацию` → тема: "способы аутентификации в веб-приложении"

### 2. Запуск агента (БЛОКИРУЮЩИЙ)

Запусти агент `deep-researcher` через Task tool:

```
subagent_type: deep-researcher
prompt: |
  ТЕМА ИССЛЕДОВАНИЯ: [тема]

  КОНТЕКСТ ПРОЕКТА: [если есть — стек, зависимости, ограничения]

  РЕЖИМ: [полный | быстрый]

  Если режим быстрый (--quick):
  - 3 поисковых запроса вместо 5
  - 5 источников вместо 8
  - Минимум 2 варианта вместо 3
  - Упрощённый отчёт без сравнительной таблицы

  Выполни полный протокол исследования: DISCOVER → FETCH → CROSS-CHECK → RANK
```

**ВАЖНО:** Используй foreground режим (НЕ background). Claude ДОЛЖЕН дождаться результата.

### 3. Представление результатов

После получения отчёта от агента:
1. Покажи полный отчёт пользователю
2. Выдели рекомендованный вариант
3. Спроси: нужно ли углубиться в какой-то вариант

### 4. Интеграция с другими скиллами

После отчёта пользователь может:
- `/smart-fix` — применить рекомендованный подход
- `/debug` — если исследование было для решения проблемы
- `/code-review` — проверить реализацию по best practices из отчёта
- Войти в Plan Mode — спланировать реализацию на основе отчёта

## Примеры

**Полное исследование:**
```
Пользователь: /research WebSocket vs SSE vs Long Polling для real-time уведомлений
Claude: Запускаю глубокое исследование...
[ждёт deep-researcher]
Claude: [показывает отчёт с 3+ вариантами, сравнительной таблицей, вердиктом]
```

**Быстрое исследование:**
```
Пользователь: /research --quick лучшая библиотека для PDF в Python
Claude: Запускаю быстрое исследование...
[ждёт deep-researcher в quick режиме]
Claude: [показывает упрощённый отчёт с 2+ вариантами]
```
