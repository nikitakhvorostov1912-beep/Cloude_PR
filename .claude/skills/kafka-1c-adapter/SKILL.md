# Kafka 1C Adapter — Интеграция 1С с Apache Kafka

Внешняя компонента для 1С:Предприятие, позволяющая публиковать и потреблять сообщения из Apache Kafka с поддержкой транзакций, AVRO, Protobuf и Schema Registry.

> Источник: github.com/NuclearAPK/Simple-Kafka_Adapter
> Платформы: Windows x64, Linux x64 (1С 8.3.24+)
> Библиотека: LibrdKafka

## Когда использовать

- Интеграция 1С с Apache Kafka (отправка/получение сообщений)
- Exactly-Once семантика для критичных операций (платежи, заказы)
- Сериализация AVRO / Protobuf для обмена данными
- Управление Schema Registry из 1С
- Мониторинг топиков и consumer groups

## Когда НЕ использовать

- Простой HTTP-обмен → используй стандартные HTTP-запросы 1С
- Обмен файлами → используй CommerceML / стандартные обмены
- RabbitMQ / другие MQ → другие компоненты

## Producer API

### Методы

| Метод | Параметры | Возврат | Описание |
|-------|----------|---------|----------|
| `УстановитьПараметр` | Имя, Значение | Boolean | Параметры librdkafka |
| `ПолучитьПараметры` | — | JSON | Все установленные параметры |
| `УстановитьПартишионер` | Тип | Boolean | До инициализации! |
| `ИнициализироватьПродюсера` | Брокеры | Boolean | Подключение |
| `ОтправитьСообщение` | Сообщение, Топик, Партиция, Ключ, Заголовки | 0/-1 | Асинхронная |
| `ОтправитьСообщениеСОжиданиемРезультата` | Сообщение, Топик, Партиция, Ключ, Заголовки | 2/1/0/-1 | Синхронная |
| `ОтправитьПакетСообщений` | JSON, Топик | Количество/-1 | Пакетная |
| `ОстановитьПродюсера` | — | Boolean | Flush + cleanup |

### Коды возврата синхронной отправки
- **2** — доставлено, подтверждено брокером
- **1** — отправлено, не подтверждено
- **0** — не доставлено
- **-1** — критическая ошибка

### Партишионеры
`consistent`, `consistent_random` (по умолчанию), `murmur2`, `murmur2_random`, `fnv1a`, `fnv1a_random`, `random`

### Пример: асинхронная отправка
```bsl
Компонента = Новый("AddIn.Integration.simpleKafka1C");
Компонента.ИнициализироватьПродюсера("broker1:9092,broker2:9092");

Для Каждого Элемент Из Данные Цикл
    Результат = Компонента.ОтправитьСообщение(Элемент.Текст, "my-topic",, Элемент.Ключ);
    Если Результат = -1 Тогда
        ВызватьИсключение Компонента.ПолучитьСообщениеОбОшибке();
    КонецЕсли;
КонецЦикла;

Компонента.ОстановитьПродюсера();
```

### Пример: синхронная с гарантией
```bsl
Компонента.УстановитьПараметр("queue.buffering.max.ms", "1");
Компонента.УстановитьПараметр("message.timeout.ms", "5000");
Компонента.ИнициализироватьПродюсера("localhost:9092");

Результат = Компонента.ОтправитьСообщениеСОжиданиемРезультата(Сообщение, "orders");
Если Результат = 2 Тогда
    // Гарантированно доставлено
КонецЕсли;
```

### Пример: пакетная отправка
```bsl
МассивСообщений = Новый Массив;
Для Каждого Запись Из Данные Цикл
    Стр = Новый Структура("message,key", Запись.Текст, Запись.ID);
    МассивСообщений.Добавить(Стр);
КонецЦикла;

ЗаписьJSON = Новый ЗаписьJSON;
ЗаписьJSON.УстановитьСтроку();
ЗаписатьJSON(ЗаписьJSON, МассивСообщений);

Отправлено = Компонента.ОтправитьПакетСообщений(ЗаписьJSON.Закрыть(), "my-topic");
```

## Consumer API

### Методы

| Метод | Параметры | Возврат | Описание |
|-------|----------|---------|----------|
| `УстановитьПараметр` | Имя, Значение | Boolean | Обязателен `group.id` |
| `ИнициализироватьКонсьюмера` | Брокер | Boolean | Подключение |
| `Подписаться` | Топики (через запятую) | Boolean | Подписка |
| `ПрочитатьСообщение` | — | Boolean | Есть ли сообщение? |
| `ЧитатьПакетСообщений` | МаксКол, Таймаут, Base64 | JSON | Пакетное чтение |
| `ОстановитьКонсьюмера` | — | Boolean | Обязателен при завершении |

### Извлечение данных (после `ПрочитатьСообщение() = Истина`)

| Метод | Возврат |
|-------|---------|
| `ПолучитьДанныеСообщения(Двоичные)` | Тело сообщения |
| `ПолучитьКлючСообщения()` | Ключ |
| `ПолучитьЗаголовкиСообщения()` | JSON массив |
| `ПолучитьТопикСообщения()` | Имя топика |
| `ПолучитьРазделСообщения()` | Номер партиции |
| `ПолучитьСмещениеСообщения()` | Offset |
| `ПолучитьВременнуюМеткуСообщения()` | Unix timestamp (мс) |

### Управление позицией
- `ПерейтиКНачалу(Топик, Партиция)` — читать с начала
- `ПерейтиККонцу(Топик, Партиция)` — только новые
- `ПерейтиКВременнойМетке(Топик, Партиция, Метка)` — к моменту времени
- `ЗафиксироватьСмещение(Топик, Смещение+1, Партиция)` — ручной коммит

### Пример: чтение сообщений
```bsl
Компонента.УстановитьПараметр("group.id", "1c-consumer-group");
Компонента.УстановитьПараметр("auto.offset.reset", "earliest");
Компонента.ИнициализироватьКонсьюмера("localhost:9092");
Компонента.Подписаться("orders,payments");
Компонента.УстановитьТаймаутОжидания(500);

Пока Истина Цикл
    Если Компонента.ПрочитатьСообщение() Тогда
        Данные = Компонента.ПолучитьДанныеСообщения(Ложь);
        Ключ = Компонента.ПолучитьКлючСообщения();
        Топик = Компонента.ПолучитьТопикСообщения();
        // Обработка...
    КонецЕсли;
КонецЦикла;

Компонента.ОстановитьКонсьюмера();
```

## Транзакции (Exactly-Once)

### Методы

| Метод | Параметры | Описание |
|-------|----------|----------|
| `ИнициализироватьТранзакционногоПродюсера` | Брокеры, УникальныйID | При рестарте — тот же ID |
| `НачатьТранзакцию` | — | Сообщения невидимы до коммита |
| `ЗафиксироватьТранзакцию` | — | Атомарный коммит (таймаут 30с) |
| `ОтменитьТранзакцию` | — | Откат, сообщения не видны |
| `ОтправитьОфсетыВТранзакцию` | — | Для Read-Process-Write |

### Когда использовать транзакции
- Критичные операции (платежи, финансы)
- Атомарные мульти-сообщения
- Read-Process-Write паттерн

### Когда НЕ использовать
- Логи, метрики (снижение throughput 20-30%)
- Простая одиночная отправка

### Пример
```bsl
Компонента.ИнициализироватьТранзакционногоПродюсера("localhost:9092", "1c-tx-producer");
Компонента.НачатьТранзакцию();

Попытка
    Компонента.ОтправитьСообщение(Данные1, "orders");
    Компонента.ОтправитьСообщение(Данные2, "audit-log");
    Компонента.ЗафиксироватьТранзакцию();
Исключение
    Компонента.ОтменитьТранзакцию();
    ВызватьИсключение ОписаниеОшибки();
КонецПопытки;
```

### Consumer для транзакционных сообщений
```bsl
Компонента.УстановитьПараметр("isolation.level", "read_committed");
Компонента.УстановитьПараметр("enable.auto.commit", "false");
Компонента.УстановитьПараметр("enable.auto.offset.store", "false");
```

| Метод | Параметры | Описание |
|-------|----------|----------|

| `ПолучитьСхемуПоИД` | URL, ID | Получение по идентификатору |

## Обработка ошибок

- **Процедуры**: при ошибке выполнение модуля прекращается
- **Функции**: возвращают `Ложь`, `-1` или пустую строку
- **Получение текста**: `Компонента.ПолучитьСообщениеОбОшибке()`
- Компонента рассчитана на серверный контекст без диалога с пользователем

### Паттерн обработки
```bsl
Результат = Компонента.ОтправитьСообщение(Данные, Топик);
Если Результат = -1 Тогда
    ТекстОшибки = Компонента.ПолучитьСообщениеОбОшибке();
    ЗаписьЖурналаРегистрации("Kafka", УровеньЖурналаРегистрации.Ошибка,,, ТекстОшибки);
КонецЕсли;
```
