# Debug — Системная отладка

Ты — опытный отладчик. Проанализируй ошибку и найди корневую причину.

## Когда использовать

- Есть ошибка, стек вызовов, или неожиданное поведение
- Тест падает
- Сервер не запускается
- Frontend показывает пустой экран или ошибки

## Когда НЕ использовать

- Для проверки качества кода → используй `/code-review`
- Для билд-ошибок → используй `build-error-resolver` агент
- Для стилистических проблем → используй `/lint-check`

## Автоматическая интеграция

Для сложных багов запусти агент `bug-hunter`:
```
subagent_type: bug-hunter
prompt: "Найди и исправь: [описание ошибки]. Стек: [стек вызовов]"
```

## Методология

### 1. Сбор информации
- Прочитай полный текст ошибки / стек вызовов
- Определи тип ошибки (runtime, syntax, logic, network, config)
- Определи где возникла (backend/frontend/build/test)

### 2. Анализ контекста
- Прочитай файл с ошибкой и связанные файлы
- Проверь импорты и зависимости
- Проверь конфигурацию (config.yaml, .env, package.json)
- Проверь версии пакетов

### 3. Воспроизведение
- Попробуй воспроизвести ошибку
- Определи минимальный сценарий воспроизведения
- Проверь: ошибка постоянная или случайная?

### 4. Диагностика
- Сужай область поиска (binary search в коде)
- Проверь граничные условия
- Проверь типы данных
- Проверь состояние файловой системы (пути, права)

### 5. Исправление
- Устрани корневую причину (не симптом!)
- Добавь обработку ошибки если нужно
- Проверь что исправление не сломало другое
- Запусти тесты

## Типичные проблемы проекта

### Backend (Python/FastAPI)
- ImportError → проверь __init__.py, круговые импорты
- Pydantic ValidationError → проверь типы в модели
- asyncio ошибки → проверь async/await
- FileNotFoundError → проверь пути (Windows: обратные слэши)
- anthropic.APIError → проверь API ключ, лимиты

### Frontend (Next.js/React)
- Module not found → проверь пути импортов, tsconfig
- Hydration mismatch → проверь "use client" директивы
- fetch failed → проверь CORS, URL бэкенда
- Type error → проверь типы в lib/types.ts

```

Причина: {корневая причина}
Файл: {путь:строка}

## Исправление
{описание что сделано}

## Проверка
{как убедиться что исправлено}
```
