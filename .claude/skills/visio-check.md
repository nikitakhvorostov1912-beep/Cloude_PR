# /visio-check — Верификация Visio диаграмм

## Назначение
Программная проверка геометрических и текстовых требований к VSDX-файлам BPMN-диаграмм.
Запускает `check_visio.py` (R1+R2) для всех `.vsdx` файлов проекта и выдаёт итоговый отчёт PASS/FAIL.

**Обязательно перед сдачей — запускать `delivery_gate.py`**, который блокирует сдачу при любых нарушениях.

## Когда использовать
- После генерации или изменения Visio диаграмм
- При подозрении на перекрытия элементов или проблемы с текстом
- Перед финальной сдачей (перед `/quality-gate`)
- После изменения констант в `layout.py`, `templates.py`, `direct_vsdx.py`

## Чеклист требований

### R1 — Геометрические (авто-проверяемые)
| Код | Требование |
|-----|-----------|
| R1.1 | Ни один bbox фигуры не пересекается с bbox другой фигуры (исключая бейджи ≤ 0.30") |
| R1.2 | Все фигуры лежат ВНУТРИ своей дорожки (swimlane) |
| R1.3 | Gap между соседними фигурами ≥ 0.03" |
| R1.4 | Ни одна фигура не выходит за границы страницы |

### R2 — Качество текста и рендеринга (авто-проверяемые)
| Код | Требование | Причина |
|-----|-----------|---------|
| R2.1 | Emoji (U+1F000+) НЕ смешаны с текстом в фигурах | Visio рендерит emoji как большие цветные иконки → "летящий" элемент |
| R2.2 | Текст doc/system-бейджей помещается в shape (нет overflow) | Иначе текст обрезается Visio |
| R2.3 | Нет символа "…" в тексте фигур | "…" = источник данных обрезал имя задачи/шлюза |

> **Важно**: R2.1 допускает чистые иконки (весь текст = один emoji, как 👤 в ролях).
> Запрещены только emoji + текст вместе (как было 📥 Название документа).

### R3 — Структурные (ручная проверка)
| Код | Требование |
|-----|-----------|
| R3.1 | Количество фигур в VSDX соответствует BPMN JSON |
| R3.2 | Стрелки соединяют правильные элементы |

## Шаги выполнения

### 1. Найти project_id
Спросить у пользователя или использовать текущий контекст.
Если не указан — проверить все проекты.

### 2. Найти все VSDX файлы
```
find backend/data/projects/<project_id>/visio/ -name "*.vsdx"
```
Типичный путь: `backend/data/projects/<uuid>/visio/proc_NNN.vsdx`

### 3. Запустить delivery_gate.py (рекомендуется) или check_visio.py

**Delivery Gate — обязательная проверка перед сдачей** (R1 + R2, блокирует при FAIL):
```bash
cd C:/CLOUDE_PR/projects/survey-automation/backend
python scripts/delivery_gate.py data/projects/<project_id>/
```

**check_visio.py — детальная проверка** (для одного файла или папки):
```bash
cd C:/CLOUDE_PR/projects/survey-automation/backend
# Один файл:
python scripts/check_visio.py data/projects/<project_id>/visio/proc_001.vsdx
# Вся папка проекта:
python scripts/check_visio.py data/projects/<project_id>/visio/
# Все проекты:
python scripts/check_visio.py data/projects/
```

### 4. Показать результаты

Показать пользователю полный вывод скрипта.

Если есть нарушения — сгруппировать по типу:
- **R1.1 перекрытия**: означает что элементы наезжают друг на друга → исправить в `direct_vsdx.py` или `layout.py`
- **R1.2 выход за дорожку**: элемент вне свой swimlane → увеличить `LANE_HEIGHT` или исправить `badge_y` в `direct_vsdx.py`
- **R1.3 малый gap**: слишком мало места между элементами → увеличить `HORIZONTAL_SPACING` / `VERTICAL_SPACING`
- **R1.4 выход за страницу**: фигура за краем страницы → увеличить размер страницы или сдвинуть элементы
- **R2.1 emoji + текст**: в badge тексте остался emoji → убрать из `_element_shapes` в `direct_vsdx.py`
- **R2.2 overflow**: текст бейджа не влезает → пересмотреть `_badge_h` коэффициенты или мин. высоту
- **R2.3 усечение "…"**: источник данных обрезал текст → добавить `_clean_text()` вызов в генератор

### 5. Итоговое заключение

Если все PASS:
```
✅ VISIO-CHECK: ВСЕ ПРОВЕРКИ ПРОШЛИ
   Файлов проверено: N
   R1.1 ✅ | R1.2 ✅ | R1.3 ✅ | R1.4 ✅
   R2.1 ✅ | R2.2 ✅ | R2.3 ✅
```

Если есть FAIL:
```
❌ VISIO-CHECK: ЕСТЬ НАРУШЕНИЯ
   Файлов с нарушениями: N

   Требует исправления:
   - R1.1: 5 перекрытий фигур → исправить layout
   - R2.1: emoji смешан с текстом → убрать из badge text
   - R2.3: усечение "…" → добавить _clean_text() в генератор

   Следующий шаг: запустить /smart-fix с описанием нарушений
```

### 6. При нарушениях — предложить исправление

Основные места для исправления:
- `backend/app/visio/direct_vsdx.py` — позиционирование конкретных элементов
- `backend/app/bpmn/layout.py` — общие константы размеров/интервалов
- `backend/app/visio/templates.py` — константы Visio в дюймах

После исправления запустить:
1. Перегенерацию: `POST /api/pipeline/generate-bpmn/<project_id>/<process_id>`
2. Повторный `/visio-check`

## Связанные инструменты
- `backend/scripts/check_visio.py` — Python скрипт (парсит VSDX XML, проверяет R1+R2)
- `backend/scripts/delivery_gate.py` — **обязательный** скрипт перед сдачей (блокирует при FAIL)
- `/quality-gate` — финальная полная проверка проекта
- `/e2e-test` — E2E тест экспорта Visio через браузер

## Примечания
- Бейджи (фигуры ≤ 0.30" × 0.30") исключены из R1.1 — они intentionally на углу задачи
- R2.1 допускает чисто-иконочные shapes (текст = только emoji, напр. 👤 ⚙) — они OK
- Заголовки дорожек (повёрнутый текст) исключены из R1.2 — у них особая геометрия
- Коннекторы (ObjType=2) исключены из всех проверок
- Файл `proc_NNN_bpmn.vsdx` — отладочный вариант через библиотеку vsdx (не основной)

## Как правильно перезапустить сервер после правок кода

uvicorn без `--reload` использует код, загруженный при старте. Нужно явно убить процесс:

```powershell
# Убить сервер на порту 8000
$conn = netstat -aon | Where-Object { $_ -match ':8000 ' -and $_ -match 'LISTENING' }
$pid8000 = ($conn.Trim() -split '\s+')[-1]
Stop-Process -Id ([int]$pid8000) -Force

# Запустить новый
Start-Process cmd -ArgumentList '/c cd /d C:\CLOUDE_PR\projects\survey-automation\backend && python -m uvicorn main:app --host 0.0.0.0 --port 8000' -WindowStyle Minimized
```

Или использовать готовый скрипт: `powershell -File C:\CLOUDE_PR\kill_restart.ps1`
