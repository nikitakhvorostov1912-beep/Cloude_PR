# Правила написания кода на 1С

> Обязательные стандарты для всех агентов, работающих с кодом 1С: `1c-code-writer`, `1c-code-reviewer`, `1c-code-simplifier`.

## Общие правила 1С-разработки

- Всегда отвечай на русском языке
- Будь брутально честным, не выдумывай факты. Честно указывай на неопределённость
- Перед ответом проверяй, полностью ли понятен запрос. Если неясно — уточни
- Предлагай и реализовывай лучшие практики, явно упоминая их

## Именование

- Не используй венгерскую нотацию: `МассивКонтрагентов` — нет, `Контрагенты` — да
- Не используй в именах переменных имена из глобального контекста 1С (Документы, Справочники, Регистры, Метаданные, Константы и др.)
- Для процедур используй глаголы: `ЗаполнитьТаблицу()`. Для функций-предикатов — вопросительную форму: `ЭтоНовый()`, `ЕстьОшибки()`

## Структура модулей

- Порядок областей в общем модуле: `ПрограммныйИнтерфейс` (public) → `СлужебныйПрограммныйИнтерфейс` (internal) → `СлужебныеПроцедурыИФункции` (private)
- Перед написанием кода проверяй в общих модулях и модулях менеджеров экспортные методы, которые можно переиспользовать

## Запросы

- Форматирование — текст без отступа на новой строке после `=`:
```bsl
Запрос.Текст =
"ВЫБРАТЬ
|	Таблица.Поле КАК Поле
|ИЗ
|	Таблица";
```
- Для результата запроса используй промежуточную переменную: `РезультатЗапроса = Запрос.Выполнить()`, не `Запрос.Выполнить().Выгрузить()`
- Всегда давай псевдонимы полям через `КАК`: `Контрагенты.ИНН КАК ИНН`
- Перед написанием запроса проверяй реквизиты метаданных (существование, имена, типы)
- Не используй запросы внутри цикла — используй пакетные запросы и временные таблицы

## Работа с данными

- Не получай реквизиты из ссылки через точку — это загружает весь объект. Используй:
  - `ОбщегоНазначения.ЗначениеРеквизитаОбъекта()`
  - `ЗначенияРеквизитовОбъекта()`
  - `ЗначениеРеквизитаОбъектов()`
  - `ЗначенияРеквизитовОбъектов()`
- При повторяющихся вычислениях в цикле используй кеширование через Соответствие

## Клиент-сервер

- В модулях форм предпочитай `&НаСервереБезКонтекста` вместо `&НаСервере`
- Минимизируй переходы клиент-сервер
- Вместо `ОписаниеОповещения` используй `Асинх`-методы где возможно

## Обработка ошибок и логирование

- Не используй `Сообщить()`. На сервере: `ОбщегоНазначения.СообщитьПользователю()`, на клиенте: `ОбщегоНазначенияКлиент.СообщитьПользователю()`
- Не глотай исключения молча — логируй через `ЗаписьЖурналаРегистрации`
- В `Попытка/Исключение` обрабатывай только ожидаемые исключения

## Антипаттерны

### Critical
- **Подзапрос в SELECT**: вложенные подзапросы в списке полей выполняются для каждой строки (N+1). Используй `ЛЕВОЕ СОЕДИНЕНИЕ` с предварительной группировкой
- **Запрос в цикле**: выноси запрос до цикла, передавай массив/список через `Ссылка В (&СписокСсылок)`
- **Обращение через точку**: загружает весь объект. Используй `ОбщегоНазначения.ЗначенияРеквизитовОбъекта()`

### High
- **Фильтр виртуальной таблицы в WHERE**: фильтруй в параметрах виртуальной таблицы, не в `ГДЕ`. Плохо: `ГДЕ Остатки.Склад = &Склад`. Хорошо: `Остатки(, Склад = &Склад)`
- **Отсутствие ПЕРВЫЕ N**: при проверке существования используй `ВЫБРАТЬ ПЕРВЫЕ 1`, не загружай всю таблицу
- **Множественные серверные вызовы**: объединяй несколько `&НаСервере` вызовов в один, возвращай Структуру с данными
- **&НаСервере вместо &НаСервереБезКонтекста**: контекстный вызов передаёт всю форму. Используй безконтекстный с явными параметрами

### Medium
- **O(n²) вложенные циклы**: при поиске по ключу — строй `Соответствие` для индексации, затем `Получить()` за O(1)
- **Глубокая вложенность >4 уровней**: используй ранний возврат (`Возврат`) и выделяй вложенную логику в отдельные функции
- **Отсутствие кеширования в цикле**: повторяющиеся одинаковые вычисления — кешируй через `Соответствие` перед циклом

### Архитектурные
- **God Module**: модуль с сотнями процедур → разделяй по ответственности на несколько общих модулей
- **Copy-Paste**: дублированный код → выноси в общие модули с экспортными методами

## Работа с чужим кодом

- При доработке чужого/типового кода оборачивай изменения в комментарии-маркеры:
```bsl
// Доработка START {Описание задачи} {Дата}
<новый код>
// Доработка END
```
- При замене существующего кода сохраняй оригинал:
```bsl
// Доработка START {Описание задачи} {Дата}
// {оригинальный код закомментирован}
<новый код>
// Доработка END
```
- Свой код (новые модули, обработки) — без маркеров, свободный рефакторинг
- Маркеры обязательны для: модулей типовых объектов, расширений (CFE), доработок БСП

## Прочее

- Для текущей даты на сервере используй `ТекущаяДатаСеанса()`, не `ТекущаяДата()`
- Проверяй код на конкурентность и необходимость блокировок. Учитывай внешние транзакции
- Ограничение строки — 120 символов (если можно корректно перенести)
- Не используй тернарный оператор `?()`
- Избегай комментариев, которые повторяют очевидное. Комментарии допускаются только для: мотивации, нетривиальных алгоритмов, ограничений, технического долга
- После редактирования кода выполняй ревью: стиль, читаемость, корректность, граничные случаи, безопасность, конкурентность

## MCP: Синтакс-помощник платформы 1С

Агенты `1c-code-writer` и `1c-code-reviewer` используют MCP-сервер `bsl-context` для проверки синтаксиса платформы.

### Доступные инструменты MCP

| Tool | Назначение | Пример |
|------|-----------|--------|
| `search` | Нечёткий поиск по API платформы | `search("МассивСтрок")` |
| `info` | Детали элемента (сигнатура, описание) | `info("Массив")` |
| `getMember` | Метод/свойство конкретного типа | `getMember("Массив", "Добавить")` |
| `getMembers` | Все методы/свойства типа | `getMembers("Массив")` |
| `getConstructors` | Конструкторы объектов | `getConstructors("Запрос")` |

### Когда использовать

- Перед написанием кода — проверить существование метода/свойства
- При ревью — валидировать сигнатуры функций платформы
- При сомнении в имени переменной — проверить коллизии с глобальным контекстом
- При работе с незнакомым типом — получить список всех методов/свойств

### Подключение

Требуется Java 17+ и платформа 1С 8.3.20+. Конфигурация в `.claude/settings.json`:
```json
{
  "mcpServers": {
    "bsl-context": {
      "command": "java",
      "args": ["-jar", "path/to/mcp-bsl-context.jar", "--platform-path", "C:/Program Files/1cv8/8.3.XX.YYYY"]
    }
  }
}
```

> Источник: github.com/alkoleft/mcp-bsl-platform-context (MIT)
