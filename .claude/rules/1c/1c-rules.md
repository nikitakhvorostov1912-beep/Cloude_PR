# Правила написания кода на 1С

> Обязательные стандарты для агентов: `1c-code-writer`, `1c-code-reviewer`, `1c-code-simplifier`.

## Общие правила

- Всегда отвечай на русском языке
- Будь брутально честным, не выдумывай факты
- Перед ответом проверяй, полностью ли понятен запрос

## Именование

- Не используй венгерскую нотацию: `МассивКонтрагентов` — нет, `Контрагенты` — да
- Не используй в именах переменных имена из глобального контекста 1С
- Для процедур используй глаголы: `ЗаполнитьТаблицу()`. Для предикатов: `ЭтоНовый()`

## Структура модулей

- Порядок областей: `ПрограммныйИнтерфейс` → `СлужебныйПрограммныйИнтерфейс` → `СлужебныеПроцедурыИФункции`
- Перед написанием кода проверяй экспортные методы общих модулей для переиспользования

## Запросы

- Форматирование — текст без отступа на новой строке после `=`
- Промежуточная переменная: `РезультатЗапроса = Запрос.Выполнить()`, не `.Выполнить().Выгрузить()`
- Всегда давай псевдонимы через `КАК`. Проверяй реквизиты метаданных перед запросом
- **Запрос в цикле — запрещён.** Используй пакетные запросы и временные таблицы

## Работа с данными

- Не получай реквизиты через точку — используй `ОбщегоНазначения.ЗначениеРеквизитаОбъекта()`
- Кеширование повторяющихся вычислений через Соответствие

## Клиент-сервер

- Предпочитай `&НаСервереБезКонтекста` вместо `&НаСервере`
- Минимизируй переходы клиент-сервер, объединяй в один вызов

## Обработка ошибок

- Не используй `Сообщить()`. Сервер: `ОбщегоНазначения.СообщитьПользователю()`, клиент: `ОбщегоНазначенияКлиент`
- Логируй через `ЗаписьЖурналаРегистрации`, не глотай исключения

## Антипаттерны

### Critical
- **Подзапрос в SELECT** → `ЛЕВОЕ СОЕДИНЕНИЕ` с группировкой
- **Запрос в цикле** → передавай через `Ссылка В (&СписокСсылок)`
- **Обращение через точку** → `ЗначениеРеквизитаОбъекта()`

### High
- **Фильтр виртуальной таблицы в WHERE** → фильтруй в параметрах таблицы
- **Без ПЕРВЫЕ N при проверке существования** → `ВЫБРАТЬ ПЕРВЫЕ 1`
- **&НаСервере вместо &НаСервереБезКонтекста** → передавай явные параметры

### Medium
- **O(n²) вложенные циклы** → строй `Соответствие` для O(1) поиска
- **Вложенность > 4 уровней** → ранний возврат, выделяй в функции

## Работа с чужим кодом

```
// Доработка START {Описание} {Дата}
<новый код>
// Доработка END
```
Маркеры обязательны для типовых объектов, расширений (CFE), БСП.

## Прочее

- Текущая дата на сервере: `ТекущаяДатаСеанса()`, не `ТекущаяДата()`
- Ограничение строки — 120 символов. Не используй тернарный оператор `?()`
- После редактирования: ревью на стиль, читаемость, корректность, безопасность

## MCP: Синтакс-помощник (bsl-context)

| Tool | Назначение |
|------|-----------|
| `search` | Нечёткий поиск по API платформы |
| `info` | Детали элемента (сигнатура, описание) |
| `getMember` | Метод/свойство конкретного типа |
| `getMembers` | Все методы/свойства типа |
| `getConstructors` | Конструкторы объектов |

Использовать: перед написанием кода, при ревью, при сомнении в имени.
Требуется Java 17+ и платформа 1С 8.3.20+.
