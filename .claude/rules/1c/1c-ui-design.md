# Правила проектирования интерфейсов 1С

> Для агентов `1c-code-writer`, `1c-code-architect` при работе с управляемыми формами.

## Элементы форм

- **Поля:** ширина под самое длинное значение. Кнопка × для очистки. Многострочный ввод для текста > 100 символов
- **Флажки:** заголовок справа. Без отрицательных формулировок. Радиокнопки: 2–5 значений, больше → список
- **Гиперссылки** — навигация. **Кнопки** — действия. Не смешивать
- **Таблицы:** числа вправо, текст влево. Пустая последняя колонка для ширины
- **Диалоги:** текст кнопки = запрашиваемое действие. Обрабатывать `КодВозвратаДиалога.Отмена`
- **Декорации:** не для заголовков полей (ломает автофокус). Изображения из библиотеки

## Компоновка форм

- **Шапка:** сверху вниз, слева направо по важности. Поля, перестраивающие форму — первыми
- **Группы:** проверять нужность заголовков. Удалять "Группа" из названий. Избегать горизонтальной компоновки
- **Сворачиваемые группы:** не помещать обязательные поля. Не размещать над обычными группами
- **Команды:** полноэкранные — основная кнопка слева сверху. Модальные — справа снизу. Одна кнопка по умолчанию
- **Условное оформление:** единый стиль, минимум цветов. Не управлять видимостью строк через него
