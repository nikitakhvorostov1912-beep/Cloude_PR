# Правила проектирования интерфейсов 1С

> Обязательные стандарты для агентов `1c-code-writer`, `1c-code-architect` при работе с управляемыми формами.
> Источник: github.com/Oxotka/1CDesignGuide (MIT)

## Элементы форм

### Поля ввода
- Ширина вмещает самое длинное ожидаемое значение
- Кнопка очистки (×) для полей с удалением/пустым значением
- Многострочный ввод для текста > 100 символов, заголовок сверху
- Не создавать поле выбора для единственного значения — заполнять автоматически
- Сохранять повторяющиеся выборы пользователя (периоды, организации)

### Флажки и переключатели
- Заголовок флажка — справа
- Избегать отрицательных формулировок ("Не запрещать...")
- Toggle — только для немедленных изменений, не более 2-3 на странице
- Радиокнопки — 2-5 значений, больше → выпадающий список
- Тумблеры — 3-5 значений, не использовать для 2

### Гиперссылки vs Кнопки
- **Гиперссылки** — навигация, открытие подчинённых форм
- **Кнопки** — действия по запросу пользователя
- Не выполнять действия через ссылки, не навигировать кнопками
- Осмысленные имена ссылок, без "тут", "нажмите", "подробнее"
- Не ставить две ссылки рядом

### Таблицы
- Заголовки колонок в одну строку
- Числа — вправо, текст — влево
- Пустая последняя колонка для управления шириной

### Диалоги
- Чёткое описание действия, однозначный вопрос
- Текст кнопки = запрашиваемое действие
- Обрабатывать `КодВозвратаДиалога.Отмена`

### Декорации
- Не использовать для заголовков полей (ломает автофокус)
- Не использовать для выравнивания — свойства групп
- Изображения — из библиотеки, не из файлов

## Компоновка форм

### Шапка
- Сверху вниз, слева направо по важности
- Поля, перестраивающие форму (Вид операции) — первыми
- Редко меняемые (Организация) — справа

### Группы
- Проверять необходимость заголовков (обычно не нужны)
- Удалять "Группа" из заголовков
- Избегать горизонтальной компоновки
- Минимальное разрешение: 1280×768px

### Сворачиваемые группы
- Описывать содержимое в заголовке
- Не помещать обязательные или часто используемые поля
- Не размещать над обычными группами

### Команды формы
- Полноэкранные: основная кнопка слева сверху ("Провести и закрыть")
- Модальные: основная кнопка справа снизу ("ОК")
- Только одна кнопка по умолчанию
- Несколько важных → жирный шрифт + фон RGB(255,255,0)

### Подвал
- Наименее важные поля: Комментарий, Ответственный

### Условное оформление
- Единый стиль для однотипных условий
- Минимум стилей и цветов
- Не управлять видимостью строк через условное оформление

## Подсказки
- Кнопочные — для первичного знакомства
- Верхние — для редко используемых форм
- Нижние — для сложных элементов
- Жёлтый фон / иконки для важных полей
