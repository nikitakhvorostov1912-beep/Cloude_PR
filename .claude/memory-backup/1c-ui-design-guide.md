# 1C Design Guide — Стандарты проектирования интерфейсов 1С

> Источник: github.com/Oxotka/1CDesignGuide (138 звёзд, MIT)

## Что это

Гайд по проектированию пользовательских интерфейсов в 1С:Предприятие. Охватывает элементы форм, компоновку, антипаттерны. Минимальное разрешение — 1280×768px.

## Элементы форм

### Декорации
- Не ставить точку в конце однострочного текста
- Не использовать декорации для заголовков полей (ломает автофокус)
- Не использовать для выравнивания — использовать свойства групп
- Изображения добавлять в библиотеку изображений, не из файлов

### Поля ввода
- Ширина должна вмещать самое длинное ожидаемое значение
- Добавлять кнопку очистки (×) для полей с возможностью удаления
- Многострочный ввод для текста > 100 символов, заголовок сверху
- Сохранять повторяющиеся выборы пользователя (периоды, организации)
- Не создавать поля выбора для единственного значения — заполнять автоматически
- Не использовать платформенный выбор типа для составных типов

### Флажки (Checkbox)
- Заголовок справа (отступая от платформенных умолчаний)
- Использовать для простых бинарных опций
- Избегать отрицательных формулировок ("Не избегать..." путает)
- Добавлять подсказку снизу если смысл неочевиден

### Переключатели (Toggle)
- Использовать когда изменение состояния происходит немедленно
- Не более 2-3 на одной странице

### Радиокнопки
- 2-5 значений; больше — выпадающий список
- Значение по умолчанию — первым
- Повторяющийся текст выносить в заголовок

### Тумблеры
- 3-5 значений, для перестроения формы или выполнения действий
- Не использовать для 2 значений (неясно что выбрано)

### Гиперссылки
- Для открытия подчинённых и сервисных форм
- Давать осмысленные, достаточно длинные имена
- Не выполнять действия через ссылки — для действий кнопки
- Избегать "тут", "нажмите", "подробнее"
- Не ставить две ссылки рядом (сложно различить)

### Кнопки
- Для действий по запросу пользователя
- Заголовок должен указывать, откроется ли новая форма
- Не использовать для навигации — для навигации гиперссылки
- Пометные кнопки — для команд строки таблицы

### Таблицы
- Заголовки колонок в одну строку
- Убирать заголовок если колонка одна
- Добавлять пустую последнюю колонку для управления шириной
- Выравнивание: числа — вправо, текст — влево

### Диалоги
- Для вопросов/предупреждений, требующих внимания
- Чёткое описание что пользователь должен сделать
- Текст кнопки повторяет запрашиваемое действие
- Обрабатывать КодВозвратаДиалога.Отмена при закрытии

### Подсказки
- Кнопочные подсказки — для первичного знакомства
- Верхние подсказки — для редко используемых форм
- Нижние подсказки — для сложных элементов
- Визуальное выделение (жёлтый фон, иконки) для важных полей

## Компоновка форм

### Группы
- Проверять необходимость заголовков групп (обычно не нужны)
- Удалять платформенное слово "Группа" из заголовков
- Избегать горизонтальной компоновки если возможно
- Типы выделения: Сильное (полоса+отступ), Обычное, Слабое

### Сворачиваемые группы
- Для скрытия элементов и сжатия форм
- Описывать содержимое в заголовке ("Банковский счёт и адрес")
- Не помещать часто используемые или обязательные поля
- Не размещать над обычными группами

### Всплывающие группы
- Для необязательных/справочных полей
- Использовать осторожно — не интуитивно для многих
- Делать заголовки описательными и привлекательными

### Командные панели
- Source=Форма для стандартных команд объекта
- Source=Глобальные для общих команд

### Команды формы
- Основная кнопка слева сверху для полноэкранных форм ("Провести и закрыть")
- Основная кнопка справа снизу для модальных форм ("ОК")
- Только одна кнопка по умолчанию
- Для нескольких важных действий: жирный шрифт + золотой фон RGB(255,255,0)

### Шапка формы
- Элементы сверху вниз, слева направо по важности
- Редко меняемые поля — справа (Организация, Подразделение)
- Поля, перестраивающие форму (Вид операции) — первыми

### Подвал формы
- Наименее важные поля или сводная информация
- Обычно: Комментарий и Ответственный

### Условное оформление
- Единый стиль для однотипных условий
- Не использовать много стилей и цветов
- Не управлять видимостью строк через условное оформление

## Ключевые принципы

**Делать:**
- Приоритет ясности и удобства пользователя
- Тестировать на минимальном разрешении 1280×768
- Осмысленные и описательные заголовки
- Группировать связанные элементы логически
- Выделять обязательные/важные поля
- Сохранять настройки пользователя

**Не делать:**
- Усложнять формы с одним элементом
- Прятать обязательную информацию в сворачиваемых группах
- Смешивать навигацию и действия (ссылки vs кнопки)
- Перегружать множеством одновременных состояний UI
- Нарушать ожидаемое поведение элементов

## Применение

Используется агентами `1c-code-writer` и `1c-code-architect` при:
- Проектировании управляемых форм
- Ревью UI решений (агент `1c-code-reviewer`)
- Создании новых форм документов и справочников
