# Протоколы обмена 1С с веб-приложениями

## CommerceML (стандарт обмена)

> Источник: https://github.com/carono/yii2-1c-exchange (Yii2-модуль, но архитектура универсальна)

### Суть протокола

CommerceML — XML-формат обмена данными между 1С и внешними системами.
Используется для синхронизации: каталог товаров, цены, остатки, заказы.

### Этапы обмена

```
1. Инициализация → авторизация (HTTP Basic Auth), проверка соединения
2. Загрузка групп → дерево категорий товаров (иерархия)
3. Загрузка свойств → характеристики товаров и их варианты
4. Загрузка типов цен → розница, опт, закупочная и т.д.
5. Импорт каталога → товары с реквизитами, свойствами, изображениями
6. Обработка предложений → варианты товаров с ценами и остатками
7. Обмен документами → заказы, статусы (опционально)
```

### Архитектура интеграции (интерфейсный подход)

Модели веб-приложения реализуют интерфейсы:

| Интерфейс | Модель | Назначение |
|-----------|--------|------------|
| GroupInterface | Категория | Дерево категорий, вложенность |
| ProductInterface | Товар | Реквизиты, свойства, изображения |
| OfferInterface | Предложение | Цены, остатки, варианты товара |
| DocumentInterface | Документ/Заказ | Позиции, статусы, контрагенты |
| PartnerInterface | Контрагент | Покупатель/поставщик |

### Ключевые методы моделей

**Статические (создание/инициализация):**
- `createModel1c()` — создать модель из данных 1С
- `createTree1c()` — построить дерево категорий
- `createProperties1c()` — создать свойства товаров
- `createPriceTypes1c()` — создать типы цен

**Экземплярные (наполнение данными):**
- `setRequisite1c()` — установить реквизит
- `setGroup1c()` — назначить группу
- `setProperty1c()` — установить свойство
- `addImage1c()` — добавить изображение
- `setPrice1c()` — установить цену

**Запросы (экспорт из веб в 1С):**
- `getIdFieldName1c()` — поле идентификатора
- `getExportFields1c()` — поля для экспорта
- `findDocuments1c()` — выборка документов для отправки

### Конфигурация обмена

```
productClass   — класс модели товара
offerClass     — класс модели предложения
documentClass  — класс модели документа
groupClass     — класс модели категории
partnerClass   — класс модели контрагента
exchangeDocuments — включить обмен заказами (bool)
useZip         — архивация данных (bool)
timeLimit      — лимит времени выполнения
memoryLimit    — лимит памяти
debug          — сохранять данные обмена для отладки
```

### Зависимости

- `zenwalker/commerceml` — парсинг XML CommerceML
- Nested sets — для эффективных иерархий категорий
- File upload — для изображений товаров

## Применимость

- Интернет-магазины с каталогом из 1С
- Синхронизация остатков/цен в реальном времени
- Двусторонний обмен заказами (сайт ↔ 1С)
- Любой веб-сервис, которому нужны данные из 1С:Управление торговлей / 1С:Комплексная автоматизация
