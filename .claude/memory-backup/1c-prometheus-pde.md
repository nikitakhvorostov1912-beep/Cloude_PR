# PDE — Prometheus Data Exporter для 1С

> Источник: github.com/freewms/PDE (126 звёзд, 73 коммита)

## Что это

Встраиваемая конфигурация 1С:Предприятие для экспорта метрик в Prometheus. Публикует HTTP-сервис, который Prometheus опрашивает по расписанию (scraping).

## Структура конфигурации

| Объект | Тип | Назначение |
|--------|-----|-----------|
| `пэмМетрики` | Справочник | Каталог метрик для мониторинга |
| `пэмСостояниеМетрик` | Регистр сведений | Текущие значения метрик |
| `пэмСостояниеМетрик` | Отчёт | Визуализация состояния |
| `Prometheus` | HTTP-сервис | Endpoint для Prometheus scraping |
| `пэмНастройкиМетрик` | Общая форма | Управление настройками |
| `пэмНастройки` | Общая команда | Команда открытия настроек |
| `Prometheus` | Подсистема | Группировка объектов |
| — | Регламентное задание | Периодический сбор метрик |
| — | Роли | Права доступа к подсистеме |
| — | Константы | Настройки подсистемы |
| — | Перечисления | Типы метрик |

## Принцип работы

1. В справочнике `пэмМетрики` настраиваются метрики (имя, тип, запрос для получения значения)
2. Регламентное задание периодически выполняет запросы и записывает значения в РС `пэмСостояниеМетрик`
3. HTTP-сервис `Prometheus` отдаёт метрики в формате Prometheus text exposition format
4. Prometheus сервер выполняет scrape по URL: `http://<server>/hs/prometheus/metrics`

## Типы метрик Prometheus

- **gauge** — текущее значение (количество сеансов, размер БД)
- **counter** — монотонно растущий счётчик (количество ошибок)
- **histogram** — распределение значений (время выполнения запросов)
- **summary** — как histogram, но с квантилями

## Примеры метрик для 1С

- Количество активных сеансов
- Количество фоновых заданий (выполняющихся / с ошибкой)
- Размер базы данных
- Время выполнения регламентных операций
- Количество документов за период
- Очередь обмена данными

## Настройка Prometheus

```yaml
# prometheus.yml
scrape_configs:
  - job_name: '1c-enterprise'
    scrape_interval: 30s
    metrics_path: '/hs/prometheus/metrics'
    static_configs:
      - targets: ['1c-server:8080']
    basic_auth:
      username: 'prometheus'
      password: 'secret'
```

## Применение

- Мониторинг производительности 1С в Grafana
- Алертинг через Prometheus Alertmanager
- Интеграция 1С в общий стек мониторинга (Prometheus + Grafana + Alertmanager)
- Отслеживание бизнес-метрик из 1С
