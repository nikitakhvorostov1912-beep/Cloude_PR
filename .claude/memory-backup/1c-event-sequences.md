# Последовательности событий объектов 1С

> Источник: https://github.com/kuzyara/Sequences-of-events-for-1C-objects

## Ключевые правила

- Все подписки на события выполняются **после** стандартных обработчиков объекта (не формы)
- Подписки выполняются **только на сервере**
- Подписки с конкретным типом источника → **раньше** подписок с общим типом (ДокументОбъект, СправочникОбъект)
- При одинаковом источнике/событии — порядок по дереву метаданных (сверху вниз)

## Проведение документа

```
1. [Сервер] Модуль объекта активируется
2. ПередЗаписью (объект)
3. → Подписки на ПередЗаписью (конкретные типы → общие)
4. ПриЗаписи (объект)
5. → Подписки на ПриЗаписи
6. ОбработкаПроведения / ПередПроведением (объект)
7. → Подписки на ОбработкаПроведения
8. ПриПроведении (объект) — формирование движений
9. → Подписки на ПриПроведении
10. Запись движений в регистры → ПередЗаписью/ПриЗаписи набора записей
```

## Отмена проведения

```
1. ПередОтменойПроведения (объект)
2. → Подписки на ПередОтменойПроведения
3. ПриОтмене (объект) — удаление движений
4. → Подписки на ПриОтмене
5. Удаление записей регистров → ПередЗаписью/ПриЗаписи набора записей
```

## Открытие формы

```
Существующий документ:
  1. [Сервер] Активация модуля
  2. ПриЧтенииНаСервере
  3. ПриСозданииНаСервере
  4. [Клиент] ПриОткрытии

Копирование:
  1. [Сервер] ПриКопировании
  2. ПриСозданииНаСервере
  3. [Клиент] ПриОткрытии

Новый документ:
  1. [Сервер] ОбработкаЗаполнения
  2. ПриСозданииНаСервере
  3. [Клиент] ПриОткрытии
```

## Запись регистра сведений (менеджер записи)

**Новая запись:**
```
ПередЗаписью → ПриЗаписи (1 раз)
```

**Модификация существующей записи (изменение ключевых полей):**
```
1. ПередЗаписью (пустой набор — удаление старой записи)
2. ПриЗаписи (пустой набор)
3. ПередЗаписью (новый набор — запись новой)
4. ПриЗаписи (новый набор)
→ Итого: ПередЗаписью и ПриЗаписи вызываются 2 раза!
```

## Обработка для анализа

`ПодпискиНаСобытияОбъектовМетаданных.epf` — показывает подписки в порядке выполнения платформой:
- Сначала конкретные типы, потом общие
- Группировка по типам объектов (документы, справочники, регистры)

## Частые ошибки

- Код в подписке зависит от порядка → убедись что подписка выше в дереве
- Изменение ключевых полей регистра → двойной вызов ПередЗаписью/ПриЗаписи
- Подписки НЕ вызываются при программной записи с `ОбменДанными.Загрузка = Истина`
