# Скилл: Линтинг и проверка типов

Запуск линтера (ruff) и проверки типов (pyright) с автоисправлением.

## Инструкции

Ты — инженер по качеству кода. Запусти автоматические проверки и исправь проблемы.

### Шаг 1: Установи инструменты (если нужно)
```bash
pip install ruff pyright
```

### Шаг 2: Запусти Ruff линтер
```bash
ruff check src/ --output-format=text
```

Автоисправление:
```bash
ruff check src/ --fix
```

Ключевые правила ruff:
- **E**: PEP8 стиль
- **F**: Pyflakes (неиспользуемые импорты, неопределённые имена)
- **W**: Предупреждения
- **I**: Сортировка импортов (isort)
- **B**: Bugbear (типичные ловушки)
- **UP**: pyupgrade (современные паттерны Python)
- **SIM**: simplify (лишняя сложность)

### Шаг 3: Форматирование
```bash
ruff format src/ --check    # Только проверка
ruff format src/             # Применить
```

### Шаг 4: Pyright
```bash
pyright src/
```

Фокус на:
- Отсутствующие аннотации типов на публичных функциях
- Несовпадение типов (str vs Path, dict vs None)
- Несовместимые return-типы
- Отсутствующие импорты

### Шаг 5: Конфиг ruff
Если `pyproject.toml` не содержит секцию `[tool.ruff]`, создай:
```toml
[tool.ruff]
target-version = "py310"
line-length = 100

[tool.ruff.lint]
select = ["E", "F", "W", "I", "B", "UP", "SIM"]
ignore = ["E501"]

[tool.ruff.format]
quote-style = "double"
```

### Шаг 6: Исправления
1. Автоисправление через `--fix`
2. Ручное исправление ошибок типов
3. Добавление type hints
4. Удаление неиспользуемых импортов
