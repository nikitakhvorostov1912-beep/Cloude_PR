# Скилл: Полный ревью проекта

Запуск ВСЕХ проверок последовательно для комплексного аудита проекта.

## Инструкции

Выполни все проверки по порядку, объединив результаты в единый отчёт:

### Фаза 1: Диагностика
Запусти `/health-check` — проверь что всё установлено и работает.

### Фаза 2: Архитектура
Запусти `/architecture-review` — анализ структуры, потока данных, связности модулей.

### Фаза 3: Поиск багов
Запусти `/find-bugs` — runtime ошибки, логические ошибки, интеграция, безопасность.

### Фаза 4: UI/UX
Запусти `/ui-ux-review` — аудит интерфейса на юзабилити и доступность.

### Фаза 5: Качество кода
Запусти `/lint-check` — линтинг ruff + проверка типов pyright.

### Итоговый отчёт

Объедини все находки в приоритизированный мастер-отчёт:

```
# Полный ревью проекта

## Краткое резюме
Здоровье системы: X/10
Архитектура: X/10
Качество кода: X/10
UI/UX: X/10
Покрытие тестами: X/10

## Критические проблемы (исправить немедленно)
1. ...

## Высокий приоритет (исправить на этой неделе)
1. ...

## Средний приоритет (бэклог)
1. ...

## Низкий приоритет (хорошо бы)
1. ...

## Рекомендуемый план действий
Неделя 1: [критические исправления]
Неделя 2: [высокий приоритет]
Неделя 3: [тесты + средний приоритет]
```

Спроси пользователя, нужно ли запустить автоматическое исправление.
