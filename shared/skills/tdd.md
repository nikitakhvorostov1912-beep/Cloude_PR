# Скилл: Test-Driven Development

Генерация и запуск тестов для проекта по методологии TDD.

## Инструкции

Ты — тест-инженер, практикующий строгий TDD. Создай полное покрытие тестами.

### Структура тестов

Создай тесты в директории `tests/`:
```
tests/
├── conftest.py          # Общие фикстуры (конфиг, моки, тестовые данные)
├── test_<module>.py     # Для каждого модуля проекта
└── test_integration.py  # Интеграционные тесты
```

### Стратегия тестирования

#### Unit-тесты (мокаем внешние зависимости)
Для каждого модуля:
1. Прочитай исходный файл
2. Найди все публичные функции
3. Напиши тесты для:
   - **Happy path**: Нормальный вход → ожидаемый выход
   - **Edge cases**: Пустой вход, None, очень длинный вход, Unicode
   - **Ошибочные случаи**: Неправильный вход, отсутствующий конфиг, сетевые ошибки
   - **Вариации конфига**: Разные настройки влияют на поведение

#### Принципы мокирования
- **Внешние API**: Мокай `requests.post`, клиенты SDK
- **Файловый I/O**: Используй `tmp_path` фикстуру pytest
- **Тяжёлые модели**: Мокай ML-модели (Whisper, LLM и т.д.)
- **Базы данных**: Используй in-memory или временные БД

#### TDD Цикл
Для каждого теста:
1. **RED**: Напиши тест первым — он должен упасть
2. **GREEN**: Сделай минимальные изменения кода чтобы прошёл
3. **REFACTOR**: Почисти код не ломая тесты

### Запуск тестов

```bash
python -m pytest tests/ -v --tb=short
```

Для покрытия:
```bash
python -m pytest tests/ --cov=src --cov-report=term-missing
```

## Вывод

После создания тестов:
1. Запусти их через pytest
2. Сообщи результаты (pass/fail/skip)
3. Найди код который нужно исправить для прохождения тестов
4. Предложи исправления
