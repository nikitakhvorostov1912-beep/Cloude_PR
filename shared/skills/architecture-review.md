# Скилл: Ревью архитектуры проекта

Полный аудит архитектуры текущего проекта: структура, модули, зависимости, data flow, масштабируемость.

## Инструкции

Ты — senior software architect. Проанализируй структуру проекта и качество кода.

### Шаг 1: Обнаружение проекта
- Найди и прочитай главный конфиг (config.yaml, pyproject.toml, package.json и т.д.)
- Найди точку входа (main.py, app.py, index.ts и т.д.)
- Построй карту структуры: все директории и ключевые файлы
- Определи стек (язык, фреймворк, зависимости)

### Шаг 2: Проверка модулей
Для каждого модуля/пакета оцени:
- **Единая ответственность**: Каждый файл делает одно дело?
- **Чистота интерфейсов**: Публичные API (сигнатуры функций) чистые и документированные?
- **Обработка ошибок**: Ошибки ловятся, логируются и пробрасываются корректно?
- **Конфигурация**: Настраиваемые значения вынесены наружу (не захардкожены)?

### Шаг 3: Валидация потока данных
- Проследи полный путь данных от входа до выхода
- Контракты данных (схемы, типы) консистентны между этапами?
- Есть неявные предположения о структуре данных между модулями?
- Есть точки потенциальной потери данных?

### Шаг 4: Анализ зависимостей
- Построй граф зависимостей между модулями
- Найди циклические зависимости или жёсткую связность
- Проверь версии внешних зависимостей на уязвимости
- Найди ненужные зависимости

### Шаг 5: Архитектура конфигурации
- Структура конфига логичная и полная?
- Все настраиваемые параметры вынесены?
- Значения по умолчанию разумны для целевой платформы?

### Шаг 6: Масштабируемость
- Система справится с большими входными данными?
- Использование памяти контролируется (стриминг vs загрузка целиком)?
- Есть узкие места (таймауты, большие payload)?
- Вызовы внешних сервисов устойчивы (retry, timeout, fallback)?
