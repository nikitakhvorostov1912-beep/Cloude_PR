# Quality Gate — Полная проверка проекта

Ты — строгий QA-инженер. Проведи полную проверку проекта Survey Automation по чек-листу приёмки.

## Порядок проверки

### 1. Запуск (критично)
- Проверь что `start.bat` существует и корректен
- Запусти backend: `cd projects/survey-automation/backend && python -m uvicorn main:app --host 0.0.0.0 --port 8000`
- Запусти frontend: `cd projects/survey-automation/frontend && npm run dev`
- Проверь http://localhost:8000/api/health отвечает 200
- Проверь http://localhost:3000 открывается

### 2. Backend API
- Запусти `cd projects/survey-automation/backend && pytest tests/ -v`
- Проверь что ВСЕ тесты зелёные
- Проверь Swagger UI: http://localhost:8000/docs
- Проверь каждый endpoint вручную через curl/httpx

### 3. Frontend Build
- Запусти `cd projects/survey-automation/frontend && npm run build`
- Не должно быть ошибок и предупреждений
- Проверь что нет TypeScript ошибок

### 4. UI Функциональность (через Preview)
Открой http://localhost:3000 через Preview и проверь:
- [ ] Главная страница загружается
- [ ] Можно создать проект
- [ ] Можно загрузить файлы (drag-drop работает)
- [ ] Все кнопки кликабельны и выполняют действие
- [ ] Навигация работает (сайдбар, ссылки)
- [ ] Нет заглушек, TODO, placeholder-контента
- [ ] Loading-состояния отображаются
- [ ] Error-состояния отображаются

### 5. Пайплайн
- Создай тестовый проект
- Загрузи тестовый транскрипт из test_data/
- Запусти все 6 стадий последовательно
- Проверь что каждая стадия завершается успешно
- Проверь что прогресс обновляется

### 6. Экспорт
- Скачай Visio (.vsdx) — проверь что файл валидный
- Скачай Word описания процессов
- Скачай Excel требования
- Скачай GAP-анализ Excel
- Скачай ZIP со всеми файлами

### 7. UX/Дизайн
- [ ] Тёмная тема по умолчанию
- [ ] Все тексты на русском
- [ ] Стиль минималистичный (Claude app)
- [ ] Нет визуальных багов

### 8. Консоль
- Проверь browser console — нет ошибок
- Проверь backend логи — нет exception

## Формат отчёта

```
# Quality Gate Report

Дата: {дата}
Статус: PASS / FAIL

## Результаты по категориям:
| Категория | Статус | Проблемы |
|-----------|--------|----------|
| Запуск | ✅/❌ | ... |
| Backend API | ✅/❌ | ... |
| Frontend Build | ✅/❌ | ... |
| UI Функциональность | ✅/❌ | ... |
| Пайплайн | ✅/❌ | ... |
| Экспорт | ✅/❌ | ... |
| UX/Дизайн | ✅/❌ | ... |
| Консоль | ✅/❌ | ... |

## Критические проблемы:
1. ...

## Рекомендации:
1. ...

```

Если хотя бы одна категория FAIL — общий вердикт FAIL. Перечисли ВСЕ найденные проблемы.
