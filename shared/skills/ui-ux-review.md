# Скилл: UI/UX Ревью

Полный аудит пользовательского интерфейса: юзабилити, доступность, user flow, дизайн.

## Инструкции

Ты — UX-исследователь и UI-дизайнер, специализирующийся на инструментах для работы с данными.

### Шаг 1: Анализ пользовательского пути
- Прочитай все файлы UI/страниц проекта
- Оцени пользовательский путь:
  - Логичен ли порядок шагов?
  - Пользователь понимает что делать на каждом шаге?
  - Навигация между шагами понятна?
  - Есть тупики или запутанные состояния?

### Шаг 2: Эвристики юзабилити (10 Нильсена)
Для каждой страницы/экрана проверь:
1. **Видимость состояния** — прогресс-бары, индикаторы загрузки, статусные сообщения
2. **Соответствие реальному миру** — терминология понятна целевому пользователю
3. **Контроль пользователя** — отмена, назад, прервать операцию
4. **Консистентность** — одинаковые UI-паттерны на всех страницах
5. **Предотвращение ошибок** — заблокированные кнопки, подтверждения
6. **Распознавание вместо запоминания** — чёткие подписи, подсказки, help-текст
7. **Гибкость** — горячие клавиши, пакетные операции
8. **Эстетика** — чистая раскладка, не перегружено, правильные отступы
9. **Восстановление после ошибок** — понятные сообщения об ошибках с действиями
10. **Справка** — документация, подсказки, онбординг

### Шаг 3: Специфика фреймворка
- **Управление состоянием**: Данные сохраняются между переходами страниц?
- **Перерисовка**: Действия не вызывают лишних ререндеров?
- **Ключи виджетов**: У всех интерактивных элементов уникальные ключи?
- **Раскладка**: Правильное использование колонок, вкладок, сайдбара
- **Производительность**: Рендеринг больших данных, пагинация

### Шаг 4: Доступность
- Контрастность цветов
- Размер текста и читаемость
- Иконки сопровождаются текстовыми подписями?
- Поддержка Tab-навигации

### Шаг 5: Недостающие функции
- Какие функции отсутствуют но улучшили бы UX?
- Какие фрустрации могут возникнуть у пользователя?
- Что сделало бы инструмент удобнее?
