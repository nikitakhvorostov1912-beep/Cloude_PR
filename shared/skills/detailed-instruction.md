# Скилл: Детальная инструкция для 1С-конфигуратора

## Когда использовать
Когда пользователь просит создать пошаговую инструкцию по работе в конфигураторе 1С или любом другом GUI-приложении, где важна точность кликов и навигации.

## Формат документа
Всегда `.docx` (Word). Генерировать через python-docx.

## Структура инструкции

### 1. Титул
- Название инструкции
- Конфигурация / система
- Дата создания

### 2. Этапы
Каждый логический блок — отдельный этап с заголовком уровня H1:
```
ЭТАП N. Название этапа
```

### 3. Внутри этапа — подразделы H2/H3
Каждый объект (реквизит, колонка, команда, элемент формы) — отдельный подраздел:
```
Реквизит 1: ИмяРеквизита
Колонка 2.3: ИмяКолонки
Команда 1: ИмяКоманды
```

### 4. Действия — нумерованный список
Каждое действие пользователя — отдельный пронумерованный пункт:
```
1. Нажмите ПРАВОЙ кнопкой мыши на "Реквизиты"
2. В контекстном меню выберите: Добавить → Реквизит
3. В свойствах справа заполните:
     Имя: ДатаНачала
     Синоним: Дата начала
```

### 5. Подпункты — с отступом
Детали внутри действия идут с отступом:
```
     Имя: значение
     Тип: значение
```

### 6. Маркеры результата (зелёные)
После завершения логического блока — строка с результатом:
```
✓ Результат: В дереве под "Реквизиты" появился элемент "ДатаНачала"
```
Формат: жирный, цвет RGB(0, 100, 0)

### 7. Маркеры предупреждений (красные)
Где легко ошибиться — предупреждение:
```
⚠ ВАЖНО: Кликайте правой кнопкой именно на "ТаблицаДокументов", а НЕ на "Табличные части"!
```
Формат: жирный, цвет RGB(139, 0, 0)

### 8. Схемы-деревья
Для проверки структуры — моноширинный блок (Consolas, 10pt):
```
Форма
    ГруппаОтбор (группа, горизонтальная)
        ДатаНачала
        ДатаОкончания
    ТаблицаДокументов (таблица)
        Документ
        Дата
```

## Правила написания

1. **Одно действие = один пункт.** Не объединять "кликните и введите" в одну строку
2. **Указывать ГДЕ кликать.** Не "добавьте реквизит", а "правой кнопкой на X → Добавить → Реквизит"
3. **Указывать ЧТО должно произойти.** После каждого действия — что пользователь увидит
4. **Называть элементы интерфейса.** "В панели Свойства справа", "на вкладке Команды внизу"
5. **Предупреждать об ошибках.** Если есть типичное место где путаются — ⚠
6. **Проверочные точки.** После каждого этапа — ✓ с описанием ожидаемого результата
7. **Типы данных — полный путь.** Не "Справочник видов РЦ", а "СправочникСсылка.ВидыРабочихЦентров"
8. **Повторяемые действия — расписывать каждое.** Даже если 5 колонок создаются одинаково — каждая отдельным подразделом

## Шаблон python-docx кода

```python
from docx import Document
from docx.shared import Pt, RGBColor
from docx.enum.table import WD_TABLE_ALIGNMENT

def add_note(doc, text):
    """Предупреждение (красное)"""
    p = doc.add_paragraph()
    run = p.add_run("⚠ " + text)
    run.bold = True
    run.font.color.rgb = RGBColor(0x8B, 0x00, 0x00)

def add_result(doc, text):
    """Результат (зелёный)"""
    p = doc.add_paragraph()
    run = p.add_run("✓ " + text)
    run.bold = True
    run.font.color.rgb = RGBColor(0x00, 0x64, 0x00)

def add_action(doc, text):
    """Пронумерованное действие"""
    return doc.add_paragraph(text, style='List Number')

def add_sub(doc, text):
    """Подпункт с отступом"""
    return doc.add_paragraph("     " + text)

doc = Document()
style = doc.styles['Normal']
style.font.name = 'Calibri'
style.font.size = Pt(11)
style.paragraph_format.space_after = Pt(4)

# Деревья структуры — моноширинный шрифт
p = doc.add_paragraph()
run = p.add_run(tree_text)
run.font.name = 'Consolas'
run.font.size = Pt(10)
```
