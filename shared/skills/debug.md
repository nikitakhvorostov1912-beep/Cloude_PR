# Скилл: Систематическая отладка

Пошаговая отладка конкретной проблемы в проекте.

## Инструкции

Ты — систематический отладчик. Следуй процессу строго.

### Шаг 1: Воспроизведение
- Пойми описание ошибки: $ARGUMENTS
- Найди файл и функцию где возникает ошибка
- Прочитай исходный код
- Проверь воспроизводимость

### Шаг 2: Изоляция
- Проследи поток данных до точки сбоя
- Проверь входные данные функции (валидны?)
- Проверь конфигурацию
- Проверь внешние зависимости (сервисы запущены? файлы существуют? место на диске?)

### Шаг 3: Анализ причины
- Найди точную строку вызывающую сбой
- Пойми ПОЧЕМУ падает (не только ЧТО падает)
- Это регрессия или новая проблема?
- Есть ли аналогичный код с таким же багом?

### Шаг 4: Исправление
- Минимальное изменение для устранения корневой причины
- Добавь обработку ошибок для предотвращения повтора
- Добавь логирование для будущей отладки
- Проверь что фикс не ломает остальное

### Шаг 5: Верификация
- Запусти сценарий снова
- Запусти связанные тесты
- Проверь edge cases вокруг исправления

## Формат вывода

```
## Отчёт об отладке

### Проблема: [описание]
### Корневая причина: [точная причина]
### Исправление: [что изменено]
### Изменённые файлы: [список]
### Верификация: [как проверить что фикс работает]
```
