# Скилл: Разработка для 1С:ERP УСО

## Конфигурация
- 1С:ERP Управление строительной организацией 2, редакция 2.5
- Платформа поддерживает синхронные вызовы

## Особенности конфигурации

### Справочники — правильные имена типов
- Подразделения: `СправочникСсылка.СтруктураПредприятия` (НЕ `СтруктурныеЕдиницы`)
- Виды рабочих центров: `СправочникСсылка.ВидыРабочихЦентров`

### Документы производства
- Ввод доступности РЦ: `Документ.ДоступностьРабочихЦентров` (НЕ `ДоступностьВидаРабочихЦентров`)
- Этапы производства: `Документ.ЭтапПроизводства2_2`

### Регистры
- Регистр накопления `ДоступностьВидовРабочихЦентров`:
  - Измерения: `ВидРабочегоЦентра`, `ДатаИнтервала`
  - Ресурсы: `Занято`, `ДоступностьПоВидуРЦ`, `ДоступностьПоРЦ`
  - Реквизиты: `МаксимальнаяДоступность`, `МаксимальнаяДоступностьПоВидуРЦ`, `КодСтрокиПродукция`, `КодСтрокиЭтапыГрафик`, `ВариантНаладки`, `ЭтоДвижениеВводаДоступности`, `ЭтоДвижениеЗаказаНаПроизводство`, `ЭтоДвижениеМаршрутногоЛиста`, `Распоряжение`, `Подразделение`, `Смена`, `КлючПартия`, `Этап`
  - Регистраторы: `ДоступностьВидаРабочихЦентров`, `ДоступностьРабочихЦентров`, `ЗаказНаПроизводство`, `КорректировкаРегистров`, `МаршрутныйЛистПроизводства`, `РегистраторГрафикаПроизводства`, `Сторно`, `ЭтапПроизводства2_2`
  - **НЕТ поля `ВидДвижения`** — не использовать в запросах
- Регистр сведений `ДоступностьРабочихЦентров`:
  - Измерения: `РабочийЦентр`, `Начало`, `Окончание`
  - Регистраторы: `ДоступностьРабочихЦентров`, `КорректировкаРегистров`

## Правила написания кода 1С для внешних обработок

### 1. Запросы к регистрам — ВСЕГДА проверять тип регистратора
Перед написанием запроса к регистру накопления — сначала выяснить через консоль запросов, КАКОЙ документ реально создаёт записи:
```
ВЫБРАТЬ ПЕРВЫЕ 10
    ТИПЗНАЧЕНИЯ(Т.Регистратор) КАК ТипРегистратора
ИЗ РегистрНакопления.<Имя> КАК Т
СГРУППИРОВАТЬ ПО ТИПЗНАЧЕНИЯ(Т.Регистратор)
```
НЕ доверять только списку регистраторов из конфигуратора — галочка стоит у многих документов, но реально записи создаёт только один-два.

### 2. Поля запросов — ТОЛЬКО из метаданных
Перед использованием поля в запросе — проверить что оно существует в конфигураторе:
- Измерения, ресурсы, реквизиты регистра
- НЕ использовать `ВидДвижения` для регистров остатков (это поле есть только у оборотных)
- НЕ угадывать имена полей — всегда спрашивать скриншот структуры

### 3. Типы данных — НЕ угадывать, спрашивать
В разных конфигурациях 1С одни и те же сущности могут называться по-разному:
- Подразделения: `СтруктурныеЕдиницы` / `СтруктураПредприятия` / `Подразделения`
- Контрагенты: `Контрагенты` / `Партнеры`
- Всегда запрашивать скриншот свойств реквизита → поле "Тип"

### 4. Диалоги — использовать синхронные вызовы
Для внешних обработок (.epf) использовать СИНХРОННЫЕ методы:
```1c
// ✅ ПРАВИЛЬНО — синхронный подход
Ответ = Вопрос("Текст вопроса", РежимДиалогаВопрос.ДаНет);
Если Ответ = КодВозвратаДиалога.Да Тогда
    // действие
КонецЕсли;
Предупреждение("Готово");

// ❌ НЕПРАВИЛЬНО — асинхронный подход (ОписаниеОповещения вызывает ошибки в обработках)
ПоказатьВопрос(Новый ОписаниеОповещения(ЭтотОбъект, "Метод", Неопределено), ...);
```
Причина: `ОписаниеОповещения` в контексте внешней обработки вызывает "Несоответствие типов (параметр номер '2')".

### 5. Формат файлов
- Инструкции: `.docx` (Word) — см. скилл detailed-instruction.md
- Код модулей: `.bsl` (1С)

## Чек-лист перед написанием кода

1. [ ] Получить скриншот структуры регистра (измерения, ресурсы, реквизиты)
2. [ ] Получить скриншот регистраторов регистра
3. [ ] Проверить через консоль запросов РЕАЛЬНЫЕ типы регистраторов в данных
4. [ ] Проверить тип реквизита "Подразделение" (может отличаться от стандарта)
5. [ ] Использовать синхронные диалоги (Вопрос/Предупреждение) вместо асинхронных
6. [ ] Не использовать поля, которых нет в метаданных (ВидДвижения и т.п.)
